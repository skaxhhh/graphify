### 한 줄 정의
AI Agent가 MCP 도구를 통해 기업 정보를 자율 수집·분석하고, 투자 인사이트를 제공하는 기업 관계 시각화 웹 서비스

### 배경
개인 투자자는 주도주를 뒤늦게 파악하거나, 기업 간의 복잡한 협력 및 투자 관계를 이해하는 데 어려움을 겪어 시장 분석에 비효율을 경험합니다.
비즈니스 분석가 또한 방대한 데이터를 수동으로 분석해야 하는 비효율성을 해소하고, 복잡하게 얽힌 기업 관계를 한눈에 파악하여 신속하고 깊이 있는 시장 통찰력을 얻고자 하는 니즈가 있습니다.
여기에 더해, Azure OpenAI 기반 AI Agent가 MCP(Model Context Protocol) 도구들을 통해 외부 데이터를 자율적으로 수집·추론하고, 그 결과를 Vector DB에 저장·검색하여 일관되고 깊이 있는 분석을 제공함으로써, 사용자 의사결정을 한층 더 고도화합니다.

### 사용자 문제
개인 투자자와 비즈니스 분석가들이 주도주 및 시장의 복잡한 기업 관계를 파악하는 데 비효율을 겪으며, 이로 인해 투자 기회를 놓치거나 시장 분석에 어려움을 겪는 문제가 있습니다. 또한 분산된 데이터 소스를 통합하여 실시간으로 인사이트를 도출하는 데 전문 지식과 시간이 많이 소요됩니다.

### 해결 방식
사용자가 특정 기업을 검색하면 AI Agent가 MCP 도구들(공시 API, 뉴스 API, 재무 데이터 API 등)을 통해 관련 데이터를 자율 수집하고, Azure OpenAI로 분석·추론하여 공급망·투자 관계·협력사를 직관적인 그래프로 시각화합니다. 분석 결과는 Vector DB에 임베딩되어 저장되며, 이력 관리 시스템을 통해 과거 분석 맥락도 반영됩니다.

### 차별점
- AI Agent의 자율적 MCP 도구 활용으로 데이터 수집 자동화
- Azure OpenAI 기반 심층 추론으로 숨겨진 관계 및 리스크 발굴
- Vector DB를 활용한 의미 기반 기업 유사도 검색 및 인사이트 재활용
- Agent 프롬프트 커스터마이징 및 도구 연결 관리 기능 제공
- 분석 이력 누적을 통한 시계열 인사이트 제공

### 타겟 사용자
1. **개인 투자자**: 특정 주도주와 연관된 공급망, 투자 관계사, 협력사 등을 빠르게 파악하여 후속 투자 기회를 찾고자 하는 개인 투자자.
2. **비즈니스 분석가/컨설턴트**: 특정 산업 또는 기업의 생태계와 복잡한 관계를 신속하게 분석하고, 경쟁사 분석·파트너십 기회 탐색·시장 진입 전략 수립 등 깊이 있는 시장 통찰력을 얻고자 하는 전문가.

### 사용 시나리오
A라는 기업이 특정 분야의 주도주로 떠오를 조짐을 보이자, 개인 투자자인 김투자 씨는 서비스에 A 기업을 검색합니다. AI Agent가 자동으로 공시 정보·뉴스·재무 데이터 MCP 도구를 호출해 데이터를 수집하고, Azure OpenAI가 추론하여 A 기업의 공급망·투자 관계·협력사를 그래프로 시각화합니다. 분석 결과는 Vector DB에 저장되어 이후 유사 기업 조회 시 참조됩니다. 김투자 씨는 AI가 도출한 인사이트 카드에서 B, C 기업의 동반 상승 가능성을 파악하고 투자를 결정합니다. 비즈니스 분석가 이분석 씨는 Agent 설정 화면에서 프롬프트를 커스터마이징하고 원하는 MCP 도구만 활성화한 뒤, D 기업의 시장 구조를 분석하여 보고서를 작성합니다.

### 제품 목표
투자자와 비즈니스 분석가가 AI Agent 기반의 자동화된 데이터 수집·분석을 통해 기업 관계를 직관적인 그래프로 파악하고, Vector DB 기반의 의미 검색과 이력 관리로 일관성 있는 심층 시장 통찰력을 신속하게 얻도록 지원합니다.

### 핵심 KPI
월간 활성 사용자(MAU) 수, 기업 검색 후 그래프 조회율, AI Agent 인사이트 수용률(사용자 피드백), 사용자당 평균 세션 시간, Vector DB 검색 정확도(임베딩 유사도 평균), Agent 도구 호출 성공률, 유료 전환율(프리미엄 기능), 사용자 만족도(NPS).

### 리스크/이슈
1. **데이터 정확성 및 최신성 부족**: 기업 관계 데이터의 방대함과 변화 속도로 인해 정확하고 최신 정보를 유지하기 어려울 수 있으며, AI 추론 오류는 사용자 신뢰도 하락으로 이어질 수 있습니다.
2. **복잡한 관계 시각화의 한계**: 너무 많은 기업이 연결될 경우 그래프가 복잡해져 직관성이 떨어지거나, 사용자 인터페이스가 불편해질 위험이 있습니다.
3. **Azure OpenAI API 비용 및 지연**: Agent가 복수의 MCP 도구를 반복 호출할 경우 API 비용 급증 및 응답 지연이 발생할 수 있습니다.
4. **Vector DB 관리 복잡성**: 임베딩 데이터 증가에 따른 인덱스 관리, 검색 성능 저하 위험이 있습니다.
5. **경쟁 서비스 등장**: 유사한 AI 기반 기능을 제공하는 경쟁 서비스가 등장할 경우 차별성 유지에 어려움이 있습니다.
6. **데이터 수집 및 활용 이슈**: 기업 관계 데이터 수집 및 활용에 있어 법적·윤리적 문제가 발생할 가능성이 있습니다.
7. **AI 환각(Hallucination) 위험**: LLM이 잘못된 기업 관계를 추론할 경우 투자 판단에 오류를 유발할 수 있으므로, 근거 출처 명시 및 면책 안내가 필수입니다.

### 사용자 역할

## 1. 기업 검색 및 기본 정보 조회

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 특정 기업의 이름을 검색하면 해당 기업의 기본 정보(업종, 주요 사업 내용 등)를 조회하고, 관계 그래프를 확인할 수 있어야 합니다.

**수용 기준:**

    1. ○ 검색창에 기업명을 입력하고 검색 버튼을 클릭하면 관련 기업 목록이 표시되어야 한다.
    2. ○ 목록에서 기업을 선택하면 해당 기업의 상세 페이지로 이동하며, 기업명, 업종, 주요 사업 내용, 최신 재무 상태(선택 사항) 등의 기본 정보가 표시되어야 한다.
    3. ○ 검색 결과가 없을 경우 '검색 결과가 없습니다' 메시지가 표시되어야 한다.
    4. ○ Vector DB 기반 의미 유사도 검색을 지원하며, 키워드가 정확하지 않아도 유사 기업을 제안할 수 있어야 한다.

### 1.1 자동 완성 검색 기능

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

검색창에 기업명 입력 시 자동 완성 기능을 제공하여 사용자가 빠르고 정확하게 원하는 기업을 찾을 수 있도록 지원한다.

#### 1.1.1 추천 검색어 노출 규칙

입력 텍스트 기준으로 기업명/종목코드/별칭 매칭 우선순위를 정의하고 상위 N개를 노출한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 1.1.2 키보드 탐색 및 선택

자동 완성 목록에서 방향키/엔터로 이동·선택이 가능해야 한다.
> **중요도**: 🟢 낮음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 1.1.3 오타/유사어 처리 (Vector 시맨틱 검색 적용)

유사 기업명(오타 포함)을 입력해도 Vector DB 임베딩 기반 유사도 검색으로 관련 기업을 제안한다. Azure OpenAI Embedding API를 활용하여 쿼리를 벡터화하고 유사 기업을 반환한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

### 1.2 검색 결과 정렬 및 필터링

검색 결과가 다수인 경우 기업명, 업종, 최신 업데이트 일자 등을 기준으로 정렬 및 필터링 기능을 제공한다.

#### 1.2.1 정렬 옵션 (기업명/업종/최신 업데이트)

검색 결과를 기업명, 업종, 최신 업데이트 일자 기준으로 정렬할 수 있어야 한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 1.2.2 필터 옵션 (업종/시장/데이터 상태)

업종, 시장 구분(선택), 데이터 최신성 상태 등으로 필터링할 수 있어야 한다.
> **중요도**: 🟢 낮음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

## 2. 체인 기업 관계 그래프 시각화

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

검색된 기업을 중심으로 AI Agent가 수집·분석한 공급망, 투자 관계, 협력사 등 다양한 유형의 체인 기업들을 직관적인 그래프 형태로 시각화합니다.

**수용 기준:**

    1. ○ 검색된 기업의 상세 페이지에서 '관계 그래프 보기' 버튼을 클릭하면 그래프 시각화 화면으로 전환되어야 한다.
    2. ○ 중심 기업을 기준으로 공급망, 투자 관계, 협력 관계 등의 연결 유형이 명확히 구분되어 시각화되어야 한다.
    3. ○ 그래프의 노드(기업)와 엣지(관계)에 마우스를 올리면 해당 기업 또는 관계에 대한 간략한 정보가 툴팁으로 표시되어야 한다.
    4. ○ 사용자는 그래프를 확대/축소, 이동할 수 있어야 하며, 특정 노드를 클릭하면 해당 기업 정보 요약을 볼 수 있어야 한다.
    5. ○ AI Agent가 분석한 관계 데이터를 기반으로 그래프를 자동 생성할 수 있어야 한다.

### 2.1 관계 그래프 화면 진입

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

기업 상세에서 관계 그래프 화면으로 진입하고, 그래프가 표시될 때까지의 로딩/오류를 처리한다.

#### 2.1.1 그래프 로딩 상태 표시

그래프 데이터 로딩 중 스피너/스켈레톤을 표시하고 사용자 조작을 제한한다. AI Agent 수집 단계별 진행 상태(예: "뉴스 수집 중...", "공시 분석 중...")를 스트리밍 방식으로 노출한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 2.1.2 그래프 오류/빈 데이터 처리

데이터가 없거나 로딩 실패 시 원인 안내와 재시도 버튼을 제공한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 2.1.3 그래프 기본 레이아웃/초기 뷰

중심 기업을 화면 중앙에 두고 기본 확대 레벨로 최초 표시한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

### 2.2 그래프 상호작용 (확대/이동/선택)

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 그래프를 확대/축소, 이동하고 노드/엣지를 선택해 정보를 확인할 수 있다.

#### 2.2.1 확대/축소 (마우스휠/버튼)

마우스휠 및 +/- 버튼으로 확대/축소가 가능해야 한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 2.2.2 드래그 이동 및 미니맵 (선택)

드래그로 그래프를 이동하고, 선택적으로 미니맵을 제공한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 2.2.3 노드 선택 시 요약 패널

노드를 클릭하면 우측 요약 패널(또는 하단 패널)에 기업 요약을 보여준다. AI Agent가 생성한 요약 텍스트와 신뢰도 라벨이 함께 표시된다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

    1. ○ Agent 실행마다 고유 세션 ID가 부여되고, 실행 단계별 로그가 기록되어야 한다.

#### 12.1.4 분석 결과 스냅샷 저장

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 13.2.1 자동 트렌드 분석 텍스트 생성

Azure OpenAI를 통해 일정 기간의 변화 데이터를 요약한 트렌드 분석 텍스트를 자동 생성한다. (예: "최근 3개월간 A 기업은 반도체 공급사와의 관계를 2건 신규 형성하였으며, 기존 협력사 1곳과의 관계가 약화되었습니다.")
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

#### 13.2.2 트렌드 기반 신호 강화

시계열 패턴에서 반복적으로 나타나는 리스크 또는 기회 신호를 강화(신뢰도 상향)하여 인사이트 카드에 반영한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: System(Agent) | **기기**: Web

## ## 3. 기술 아키텍처 개요 (AI Agent)

### 전체 시스템 구성

```
[사용자 브라우저]
      |
[Frontend (React/Next.js)]
      |
[API Gateway / BFF]
      |
      +---> [기업 데이터 서비스]
      |           |
      |           +---> [관계형 DB: 기업 정보, 관계 데이터]
      |           +---> [Vector DB: Azure AI Search / Qdrant (임베딩 저장)]
      |
      +---> [AI Agent 오케스트레이터]
                  |
                  +---> [Azure OpenAI (GPT-4o): 분석·추론·요약]
                  |
                  +---> [Azure OpenAI Embedding (text-embedding-3-large)]
                  |
                  +---> [MCP Tool Runner]
                              |
                              +---> [MCP 도구 1: 공시 정보 API (DART/금감원)]
                              +---> [MCP 도구 2: 뉴스 API]
                              +---> [MCP 도구 3: 재무 데이터 API]
                              +---> [MCP 도구 4: 주가 데이터 API]
                              +---> [MCP 도구 N: 기타 외부 API]
```

### AI Agent 실행 흐름

```
1. 사용자 기업 검색 요청
2. Agent 오케스트레이터: 태스크 계획 수립 (Azure OpenAI)
3. MCP Tool Runner: 필요 도구 순차/병렬 호출
   - 공시 정보 수집
   - 뉴스 수집
   - 재무 데이터 수집
4. Vector DB 조회: 유사 과거 인사이트 검색 (RAG)
5. Azure OpenAI: 수집 데이터 + RAG 컨텍스트 기반 분석·추론
6. 분석 결과: 관계 그래프 데이터, 인사이트 카드, 신호 목록 생성
7. 결과 임베딩 → Vector DB 저장
8. 실행 로그 → 이력 DB 저장
9. 사용자에게 결과 반환 (스트리밍)
```

### 주요 기술 스택 (권장)
- **Frontend**: React / Next.js, D3.js 또는 React Flow (그래프 시각화)
- **Backend**: Node.js / Python FastAPI
- **AI Engine**: Azure OpenAI (GPT-4o, text-embedding-3-large)
- **Vector DB**: Azure AI Search (벡터 인덱스) 또는 Qdrant
- **Agent 프레임워크**: LangChain / Semantic Kernel (MCP 도구 오케스트레이션)
- **관계형 DB**: PostgreSQL
- **캐시**: Redis
- **로그 수집**: Azure Monitor / Application Insights
- **인증**: OAuth 2.0 (Google/Naver/Kakao), JWT
