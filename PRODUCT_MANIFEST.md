## 1. Product Summary

### 1.1 제품의 목적
룰 기반 매매 전략을 **백테스트로 검증 → 실시간 모의(PAPER_LIVE)로 성과 확인 → 토스증권 실계좌(LIVE)로 승격**하는 일관된 파이프라인을 하나의 웹 콘솔에서 제공한다. 사용자는 코드 없이 폼/드롭다운으로 전략을 정의하고, 5분봉 인트라데이 데이터로 검증하며, 자동 평가·가상 체결 봇의 동작을 실시간으로 모니터링한다.

### 1.2 핵심 사용자
- **개인 트레이더(주 사용자)**: KOSPI 종목 대상 자동매매 전략을 직접 만들고 검증·운영하려는 개인. 코딩 비전문가도 폼으로 룰 구성 가능.
- (역할 분화는 §2 참조. 현재 제품은 단일 사용자 콘솔 전제.)

### 1.3 주요 사용 시나리오
1. **전략 만들기**: 룰 빌더에서 유니버스·진입/청산 조건·사이징·제약을 폼으로 구성하고 저장(DRAFT).
2. **검증하기**: 백테스트로 5분봉 인트라데이 기간을 돌려 수익곡선·캔들·지표·거래근거 확인.
3. **운영하기**: 룰을 ACTIVE로 전환 후 PAPER_LIVE 실행(start) → 봇이 5분 주기로 자동 평가·가상 체결.
4. **관찰하기**: 대시보드(잔고/포지션), 모니터(신호 로그/체결 피드), 거래 이력(근거+캔들), 성과 리포트로 결과 추적.
5. **승격하기(미래)**: 토스 자격증명 등록 → 검증된 룰을 LIVE로 승격해 실주문 발행(Phase 8).
6. **외부 신호 연동(미래)**: TradingView webhook 신호를 룰에 연결(Phase 7).

## 2. User Roles

> plan.md는 명시적 역할 분리를 정의하지 않음(단일 사용자 콘솔). 아래는 plan.md에서 추론 가능한 범위. (§11-A1)

| 역할 | 설명 | 주요 권한 / 기대 행동 |
|------|------|----------------------|
| **트레이더(기본 사용자)** | 전략 생성·검증·운영 주체 | 룰 CRUD, 백테스트 실행, ACTIVE 전환, PAPER_LIVE 시작/중지, 대시보드·모니터·리포트 조회, 토스 자격증명 등록, 모드(PAPER/LIVE) 전환 |
| **(미래) LIVE 운영자** | 실계좌 승격·실주문 책임 | PAPER_LIVE→LIVE 승격(토스 인증+5거래일 게이트), 실주문 모니터링, 서킷 브레이커 인지 |

- 모드 전환·LIVE 승격은 **위험 동작**으로, 확인 게이트를 거친다(역할 권한이 아니라 확인 플로우로 통제).
- 별도 관리자(Admin) 화면은 본 Trading 콘솔 범위 밖(별도 `/admin`).

## 3. Core User Goals

1. **코드 없이 매매 전략을 정의한다** — 폼/드롭다운으로 완전한 룰 구성.
2. **전략이 과거에 통했는지 빠르게 검증한다** — 백테스트 지표·수익곡선·캔들·거래근거.
3. **검증된 전략을 자동으로 굴린다** — ACTIVE 전환 후 PAPER_LIVE 자동 평가·가상 체결.
4. **봇이 지금 무엇을 하는지 안다** — 실시간 신호 로그·체결 피드·시장/스케줄러 상태.
5. **내 모의 계좌가 어떤 상태인지 안다** — 잔고·포지션·손익·성과 추이.
6. **각 매매가 왜 일어났는지 이해한다** — 거래별 트리거 조건·지표값 근거.
7. **(미래) 검증된 전략을 실제 돈으로 운영한다** — 토스 연동·LIVE 승격·실주문.
8. **(미래) 외부 신호(TradingView)를 전략에 연결한다**.

## 4. Information Architecture

### 4.1 1차 분기 축 — 트레이딩 모드
전체 IA는 **PAPER vs LIVE 모드**로 분기한다. 모드는 전역 상태이며, 사이드바 토글로 전환(LIVE 진입 시 위험 확인 모달). 현재 모드는 항상 화면에 표시된다. 모드와 다른 경로 접근 시 해당 모드 기본 화면으로 리다이렉트(ModeGuard).

### 4.2 메뉴 그룹 구조 (개편 후 통합 IA 제안)

```
Trading 콘솔 (사이드바: graphify 트레이딩 봇 · BETA)
│
├── [공통] (모드 무관, 항상 표시)
│   ├── DDS Agent (챗)            /trading                 ← 진입 index
│   └── 토스 설정                  /trading/settings
│
├── [PAPER 모드]
│   ├── 모의 대시보드              /trading/paper/dashboard
│   ├── 전략 설정 (룰 CRUD)        /trading/paper/rules
│   │     └── 룰 빌더(new/edit)    /trading/paper/rules/new · /edit/:id
│   ├── 전략 운영 (실행 제어)       /trading/paper/rules-lifecycle
│   ├── 백테스트                   /trading/paper/backtest
│   ├── 동작 모니터링              /trading/paper/monitor  ← (재배치: 현재 LIVE에 있으나 PAPER 데이터)
│   ├── 모의 거래 이력             /trading/paper/history
│   └── 모의 성과 리포트           /trading/paper/report
│
├── [LIVE 모드] (Phase 8에서 실데이터화, 현재 일부 스텁)
│   ├── 대시보드                   /trading/dashboard       🚧 stub
│   ├── 전략 설정/운영 (PAPER와 룰 공유)
│   ├── 동작 모니터링              /trading/monitor
│   ├── 거래 이력                  /trading/history         🚧 stub
│   └── (LIVE 승격 게이트 · 서킷 브레이커 경고 — Phase 8 슬롯)
│
└── [하단] 모의/실거래 토글 · "메인으로"
```

### 4.3 화면 간 관계 / 주요 경로
- **진입**: `/trading` → DDS Agent 챗(index). 사이드바로 각 화면 이동.
- **전략 생성 흐름**: 전략 설정 → "+ 새 룰" → 룰 빌더 → 저장 → 전략 설정 목록 복귀.
- **검증 흐름**: 백테스트(룰 선택·실행) → 결과(차트/캔들/거래) → 거래 행 클릭 → 근거·캔들.
- **운영 흐름**: 전략 설정(활성화) → 전략 운영(시작) → 동작 모니터링/대시보드/거래 이력 관찰.
- **승격 흐름(미래)**: 토스 설정(자격증명) → 전략 운영(LIVE 승격 게이트) → LIVE 대시보드.
- **이탈**: 하단 "메인으로"(`/` 메인 앱). 다크 테마 해제.
- **교차 링크**: 모의 대시보드 → (실계좌 미연동 시) 토스 설정. 백테스트/전략운영 → 빈 유니버스 시 종목 선택 모달.

## 5. Screen Manifest

> 각 화면: Purpose / Primary Users / Required Features / Primary·Secondary Actions / Required Data / User Inputs / System Outputs / Navigation / Priority. 상태는 §7, 상호작용은 §8, 데이터 표현은 §9, 레이아웃은 §10 참조.

### 5.1 DDS Agent (챗)
- **Purpose**: 봇 상태 조회·거래이력 요약·룰 설명·리포팅을 대화형으로 제공하는 에이전트 인터페이스.
- **Primary Users**: 트레이더.
- **Required Features**: 대화 메시지 영역(user/assistant 버블), 입력창(자동 높이, Enter 전송·Shift+Enter 줄바꿈), 타이핑 인디케이터, 에이전트 상태 표시(대기 중/응답 중).
- **Primary Actions**: 메시지 전송.
- **Secondary Actions**: (추론) 대화 스크롤, 입력 멀티라인.
- **Required Data**: 대화 메시지 목록(역할·내용·시각), 에이전트 연결 상태.
- **User Inputs**: 자유 텍스트 질의.
- **System Outputs**: 에이전트 응답 메시지. *(현재 백엔드 미연동 목업 — 실연동 전제로 설계, §11-A2)*
- **Navigation**: `/trading`(index). 모드 무관 공통.
- **Priority**: **Should** (화면 골격은 MVP, 실연동은 Should/Later).

### 5.2 토스 설정
- **Purpose**: 토스증권 Open API 자격증명 등록 + 토큰 상태 관리.
- **Primary Users**: 트레이더 / (미래)LIVE 운영자.
- **Required Features**: 연결 상태 카드(상태 배지·토큰 만료시각), 자격증명 폼(Client ID/Secret), 토큰 수동 갱신, 암호화·발급 경로 안내.
- **Primary Actions**: 자격증명 저장.
- **Secondary Actions**: 토큰 수동 갱신.
- **Required Data**: 연결 상태(미설정/설정됨·토큰 유효/설정됨·토큰 만료), 토큰 만료시각.
- **User Inputs**: Client ID, Client Secret (둘 다 마스킹 입력).
- **System Outputs**: 저장/갱신 성공·실패 메시지, 갱신된 상태 배지.
- **Navigation**: `/trading/settings`. 공통 메뉴. 모의 대시보드 실계좌 섹션에서 유입.
- **Priority**: **MVP**.

### 5.3 모의 대시보드
- **Purpose**: 모의 계좌 현황(잔고·포지션·손익·활성 룰)을 한눈에.
- **Primary Users**: 트레이더.
- **Required Features**: 통계 카드 4종, 미실현 손익 합계, 보유 포지션 테이블, 토스 실계좌 잔고(접이식), 30초 자동 갱신.
- **Primary Actions**: (조회 중심) 자동 갱신.
- **Secondary Actions**: 토스 실계좌 잔고 펼치기/접기, (미연동 시) 토스 설정 이동.
- **Required Data**: 총 평가금액(+수익률), 가용 현금, 오늘 실현손익, 활성 PAPER_LIVE 룰 개수, 미실현 손익 합계, 포지션 목록(종목/수량/평균단가/현재가/평가금액/평가손익/손익률), 토스 계좌별 잔고/출금가능.
- **User Inputs**: 없음.
- **System Outputs**: 카드·테이블·접이식 섹션, 손익 색상(양/음/0).
- **Navigation**: `/trading/paper/dashboard`. PAPER 진입 시 기본 화면.
- **Priority**: **MVP**.

### 5.4 모의 거래 이력
- **Purpose**: 체결된 모의 거래 내역 + 매매 근거 + 캔들 검증.
- **Primary Users**: 트레이더.
- **Required Features**: 캔들 차트 섹션(선택 거래의 5분봉·진입/청산 마커·하이라이트), 거래 테이블, 행 클릭 시 매매 근거 아코디언 + 해당일 캔들 전환, 30초 자동 갱신.
- **Primary Actions**: 거래 행 선택(→ 캔들 전환 + 근거 펼침).
- **Secondary Actions**: 근거 아코디언 접기.
- **Required Data**: 거래 목록(체결시각/종목/구분/수량/가격/손익/근거 JSON), 선택 거래의 5분봉 캔들, 근거(청산 사유·조건별 ✓/✗·지표값).
- **User Inputs**: 거래 행 클릭.
- **System Outputs**: 캔들 차트, 거래 테이블, 근거 상세.
- **Navigation**: `/trading/paper/history`.
- **Priority**: **MVP**.

### 5.5 전략 설정 (룰 CRUD) — `PaperRulesPage`
- **Purpose**: 룰 CRUD + **설정 축(DRAFT↔ACTIVE)** 관리. 실행 제어 미노출.
- **Primary Users**: 트레이더.
- **Required Features**: 룰 목록 테이블(이름/설정 상태 배지/쿨다운/수정일/관리), 새 룰 생성 진입, 행별 활성화↔하향·편집·복제·삭제.
- **Primary Actions**: "+ 새 룰", 활성화 / 하향.
- **Secondary Actions**: 편집, 복제, 삭제.
- **Required Data**: 룰 목록(id/이름/configStatus/runStatus/쿨다운 봉수/수정일).
- **User Inputs**: 행 액션 버튼 클릭.
- **System Outputs**: 상태 배지(DRAFT/ACTIVE), 작업 결과/실패 배너. 가드: RUNNING이면 하향·편집 불가(툴팁 안내).
- **Navigation**: `/trading/paper/rules` ↔ 룰 빌더(new/edit). LIVE 모드에서도 룰 공유.
- **Priority**: **MVP**.

### 5.6 전략 운영 (실행 제어) — `TradingRulesPage`
- **Purpose**: **ACTIVE 룰만** 표시 + **실행 축(STOPPED↔RUNNING)** 제어. 설정 변경 미노출.
- **Primary Users**: 트레이더.
- **Required Features**: ACTIVE 룰 목록(이름/실행 상태 배지/수정일/제어), 시작↔중지, 복사, 실시간 랭킹 실패 시 종목 직접 선택 모달 폴백.
- **Primary Actions**: 시작 / 중지.
- **Secondary Actions**: 복사, (미래)LIVE 승격.
- **Required Data**: ACTIVE 룰 목록(runStatus), (미래)승격 가능 여부·운영 일수.
- **User Inputs**: 시작/중지 클릭, (폴백) 종목 직접 선택.
- **System Outputs**: 실행 상태 배지(실행 중/중지됨), 작업 실패 배너, 종목 선택 모달.
- **Navigation**: `/trading/paper/rules-lifecycle`(PAPER) · `/trading/rules`(LIVE).
- **Priority**: **MVP** (LIVE 승격 게이트는 Later).

### 5.7 룰 빌더 (새 룰 / 편집) — `TradingRulesEditPage`
- **Purpose**: JSON 없이 폼으로 완전한 RuleDefinition 구성. 빌더↔JSON 양방향.
- **Primary Users**: 트레이더.
- **Required Features**: 탭(빌더/JSON), 섹션 6종 — ①룰 이름 ②유니버스(직접 종목/거래량 상위 N) ③진입 조건(논리+조건행 N) ④청산 조건(익절%/손절%+지표 조건행 N) ⑤사이징(현금/비중/수량) ⑥제약(쿨다운/최대 포지션). JSON 직접 편집.
- **Primary Actions**: 저장.
- **Secondary Actions**: 조건 추가/삭제, 탭 전환, 취소, 돌아가기.
- **Required Data**: (편집 시) 기존 룰 정의·이름. 지표/연산자/유니버스/사이징 옵션 목록.
- **User Inputs**: 이름, 유니버스 타입·종목·topN·추가종목, 진입/청산 조건(지표·기간·연산자·값/지표), 사이징 타입·값, 쿨다운·최대 포지션, (JSON 탭) 원문.
- **System Outputs**: 직렬화된 RuleDefinition, 검증 오류(formError/jsonError) 인라인.
- **Navigation**: `/trading/paper/rules/new` · `/edit/:id`(+ LIVE `/trading/rules/edit`). 저장 후 전략 설정 복귀.
- **Priority**: **MVP**.

### 5.8 백테스트 — `PaperBacktestPage`

## 7. State Requirements

> 공통: 로딩=스켈레톤/플레이스홀더 텍스트, 에러=인라인 메시지, 빈=안내 문구 + 다음 행동 유도. 손익 데이터는 양(+/녹색)·음(-/적색)·0(회색) 색상 구분.

### 7.1 DDS Agent 챗
- Default: 환영 메시지 1개 + 입력창.
- Empty: (대화 없음 = 환영 메시지가 기본).
- Loading: 타이핑 인디케이터(점 3개 애니메이션).
- Error: (추론) 응답 실패 메시지 버블.
- Success: 에이전트 응답 버블 추가, 하단 자동 스크롤.
- Disabled: 입력 비었거나 응답 중일 때 전송 버튼 비활성.

### 7.2 토스 설정
- Default: 상태 카드 + 빈 자격증명 폼.
- Loading: 상태 "확인 중...".
- Error: 저장/갱신 실패 메시지(적색).
- Success: "자격증명이 저장되었습니다"/"토큰이 갱신되었습니다"(녹색), 배지 갱신, 입력 초기화.
- Disabled: ID/Secret 미입력 시 저장 버튼 비활성, 갱신 중 갱신 버튼 비활성.
- Permission/상태 분기: 미설정 / 설정됨·토큰 유효 / 설정됨·토큰 만료 배지.

### 7.3 모의 대시보드
- Default: 카드 4 + 미실현 합계 + 포지션 테이블 + 실계좌 접이식.
- Empty: 포지션 "보유 포지션 없음"; 실계좌 미연동 "토스 미연동 — 토스 설정 유도".
- Loading: 스켈레톤(카드 4 + 블록).
- Error: "대시보드 데이터를 불러올 수 없습니다."
- Success: 30초마다 데이터 갱신.

### 7.4 모의 거래 이력
- Default: 캔들 섹션 + 거래 테이블, 첫 거래 자동 선택.
- Empty: "체결된 모의 거래가 없습니다."
- Loading: "불러오는 중...".
- Error: "거래 이력을 불러오지 못했습니다."
- Success: 행 선택 시 캔들 전환 + 근거 아코디언(근거 없으면 "근거 정보 없음").

### 7.5 전략 설정 (룰 CRUD)
- Default: 룰 테이블 + "+ 새 룰".
- Empty: "등록된 룰이 없습니다. + 새 룰로 추가하세요."
- Loading: "불러오는 중...".
- Error: "룰 목록을 불러오지 못했습니다." / 작업 실패 배너.
- Success: 활성화/하향/복제/삭제 후 목록 갱신.
- Disabled: RUNNING 룰 → 하향·편집 비활성(툴팁: "전략 운영에서 먼저 중지").

### 7.6 전략 운영
- Default: ACTIVE 룰 테이블.
- Empty: "ACTIVE 상태인 룰이 없습니다. 전략 설정에서 활성화하세요."
- Loading/Error: 동일 패턴 + 작업 실패 배너.
- Success: 시작/중지/복사 후 갱신.
- 특수: 실시간 랭킹 실패 → 종목 직접 선택 모달.

### 7.7 룰 빌더
- Default(새 룰): 기본값 폼(예시 종목·기본 조건 1행).
- Default(편집): 기존 룰 역직렬화 로드.
- Loading: 편집 모드 "불러오는 중...".
- Error: formError(이름/조건/사이징 검증), jsonError("JSON 형식이 올바르지 않습니다").
- Success: 저장 후 전략 설정 이동.
- Disabled: 저장 중 버튼 비활성.

### 7.8 백테스트
- Default: 입력 폼만(결과 영역 비어 있음).
- Empty(결과): 거래 0건 "체결된 거래가 없습니다."
- Loading: 버튼 "실행 중...".
- Error: 인라인 에러 / 빈 유니버스 → 종목 선택 모달.
- Success: 지표·차트·통계·거래 테이블 렌더, 첫 거래 자동 선택.
- Disabled: 룰 미선택 시 실행 비활성.

### 7.9 모의 성과 리포트
- Default/Success: 수익곡선 + 6지표 카드.
- Empty: "모의 실행 데이터가 없습니다. PAPER_LIVE 룰 활성화 시 자동 기록."
- Loading/Error: 표준 패턴.

### 7.10 동작 모니터링
- Default/Success: 상태 행 + 신호 로그 + 체결 피드 + 마지막 갱신 시각.
- Empty: 신호 "최근 신호 없음", 체결 "오늘 체결 내역 없음".
- Loading/Error: 표준 패턴.

### 7.11 LIVE 화면(미래)
- 현재: "준비 중" 플레이스홀더.
- 미래(Phase 8): PAPER 대응 상태 + 서킷 브레이커 경고(open 시 평가 중단 배너), 주문 실패 상태.

## 8. Interaction Requirements

### 8.1 생성 플로우 (룰)
전략 설정 "+ 새 룰" → 룰 빌더(폼/JSON) → 검증(이름·진입조건≥1·사이징) → 저장 → 전략 설정 목록 복귀(신규 DRAFT).

### 8.2 수정 플로우 (룰)
전략 설정 행 "편집"(ACTIVE+RUNNING이면 비활성) → 룰 빌더 역직렬화 로드 → 수정 → 저장. *(미래 RULE-05: LIVE 룰은 직접 편집 불가 → DRAFT 복사본 안내.)*

### 8.3 삭제 플로우 (룰)
전략 설정 행 "삭제" → (확인 절차 권장, §11-A4) → 목록에서 제거.

### 8.4 상태 전이 플로우 (룰 2축)
- 설정 축: DRAFT --활성화--> ACTIVE --하향(RUNNING 아닐 때)--> DRAFT.
- 실행 축: (ACTIVE) STOPPED --시작--> RUNNING --중지--> STOPPED.
- 복제/복사: 어느 화면에서나 동일 정의의 DRAFT 사본 생성.

### 8.5 검색 / 필터 / 정렬 플로우
- 현행: 명시적 검색/정렬 없음. 전략 운영은 ACTIVE만 필터링(고정).
- 종목 선택 모달(`CompanyPickerModal`): 종목 검색·다중 선택(빈 유니버스 폴백).
- (미래 Later: 리포트 기간·룰 필터, 거래 이력 페이지네이션/정렬.)

### 8.6 선택 / 확인 / 취소 플로우
- 거래 행 선택: 백테스트·거래 이력에서 행 클릭 → 캔들 전환 + 근거 펼침/접힘 토글.
- 모드 전환 확인: PAPER→LIVE 시 확인 모달(취소/실거래로 전환).
- 룰 빌더 취소: "취소"/"← 돌아가기" → 변경 폐기 후 목록.
- (미래) LIVE 승격: 게이트 충족 확인 → 승격 확인.

### 8.7 알림 / 토스트 / 모달 사용 상황
- **모달**: ① PAPER→LIVE 위험 확인 ② 종목 직접 선택(백테스트 빈 유니버스 / 전략운영 랭킹 실패).
- **인라인 배너/메시지**: 작업 실패(룰 활성화/시작/저장), 토스 저장·갱신 결과, 폼 검증 오류.
- **자동 갱신 표시**: 대시보드/모니터/이력 "30초 자동 갱신" + 마지막 갱신 시각.
- (미래) 서킷 브레이커 열림 경고 배너(모니터/LIVE 대시보드).
- *현행 전역 토스트 컴포넌트 사용 여부는 §12 확인 필요.*

### 9.1 반드시 표시되어야 하는 데이터 (화면별 핵심)
- **모의 대시보드**: 총 평가금액·가용 현금·오늘 실현손익·활성 PAPER_LIVE 룰 수·미실현 손익 합계·포지션(종목/수량/평균단가/현재가/평가금액/평가손익/손익률)·토스 계좌 잔고/출금가능.
- **거래 이력**: 체결시각·종목(회사명+코드)·구분(매수/매도)·수량·가격·손익·근거(청산 사유·조건 ✓/✗·지표값)·5분봉 캔들.
- **전략 설정**: 룰 이름·설정 상태·쿨다운(봉+분 환산)·수정일.
- **전략 운영**: 룰 이름·실행 상태·수정일.
- **백테스트**: 최종 평가액·수익률·MDD·승률·거래수·Sharpe·Sortino·Profit Factor·수익곡선·드로우다운 구간·거래 목록·캔들.
- **성과 리포트**: 기간·수익곡선·총수익률·MDD·승률·총거래·Sharpe·Sortino.
- **모니터링**: 시장 상태·스케줄러 최근 실행·신호(시각/종목/신호/체결/RSI14/SMA20/현재가)·오늘 체결(시각/종목/구분/수량/체결가/손익).
- **토스 설정**: 연결 상태·토큰 만료시각.

### 9.2 사용자가 입력해야 하는 데이터
- **룰 빌더**: 이름, 유니버스(직접 종목 코드 / topN+추가종목), 진입 조건(지표·기간·연산자·값|지표·논리), 청산(익절%·손절%·지표 조건), 사이징(타입·값), 제약(쿨다운·최대 포지션). (JSON 탭: 원문.)
- **백테스트**: 룰 선택, 시작일·종료일, 시작·종료 시각, 초기 자본, (폴백)종목.
- **토스 설정**: Client ID, Client Secret.
- **DDS Agent**: 자유 텍스트.
- **종목 선택 모달**: 종목 검색·선택.

### 9.3 검증이 필요한 데이터
- 룰: 이름 필수 · 진입 조건 ≥1 · 사이징 값 숫자 · (지표별)기간 · JSON 파싱 유효성.
- 백테스트: 룰 선택 필수 · 기간(최대 60일·5분봉 제약) · 시간대(HH:mm) · 초기자본 숫자.
- 토스: Client ID/Secret 미입력 시 저장 차단.
- (미래) LIVE 승격: 토스 인증 완료 + 최소 5거래일 운영.

### 9.4 적절한 표현 방식
| 데이터 | 표현 |
|--------|------|
| 핵심 지표(평가금액·수익률·Sharpe 등) | **통계 카드** (라벨+값+보조) |
| 포지션·거래·신호·룰 목록 | **테이블** (행 액션/행 클릭 확장) |

| 대화 | **메시지 버블 + 입력창** |

## 11. Assumptions

- **A1 (역할)**: plan.md는 단일 사용자 콘솔만 정의. "LIVE 운영자"는 동일 사용자의 권한 강화 단계로 가정(별도 계정 분리 아님). RBAC 미정.
- **A2 (DDS Agent 연동)**: "실연동 전제"는 화면 골격을 실연동 가능 구조로 설계한다는 의미. 실제 Agent API 스펙·기능 범위는 미정 → 화면은 메시지/입력/상태 패턴으로 일반화.
- **A3 (모니터 모드 귀속)**: 동작 모니터링이 LIVE 메뉴에 있으나 PAPER 데이터 사용. 개편 시 **PAPER 그룹으로 이동 또는 모드별 데이터 분기**로 가정(plan.md §6-④ 권고 반영). 최종 귀속은 디자인 결정.
- **A4 (삭제 확인)**: 룰 삭제 시 확인 절차가 현행 코드엔 없으나, 파괴적 동작이므로 확인 모달/인라인 확인 배치를 가정.
- **A5 (테이블 정렬/페이지네이션)**: 현행 미구현. MVP는 단순 목록, 페이지네이션/정렬은 Later(REPORT-03 등)로 가정.
- **A6 (다크 테마)**: Trading 콘솔은 다크 테마 전용(코드상 `bg-gray-950` 등). 개편 시 유지하되 shared 토큰으로 정렬 가정.
- **A7 (LIVE 룰 공유)**: LIVE 모드의 전략 설정/운영은 PAPER와 동일 룰 데이터를 공유(현행 컴포넌트 재사용)한다고 가정. LIVE 전용 룰 분리 여부는 미정.
- **A8 (자동 갱신 주기)**: 대시보드·모니터·이력 30초, 폴링 방식(WebSocket 제외) 유지 가정.
- **A9 (반응형)**: 데스크탑 우선 + 모바일 드로어. 테이블은 모바일에서 가로 스크롤 또는 카드 전환 가정.
