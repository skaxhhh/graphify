# Phase 6.5 Plan 05: Frontend Role Split + Rationale Accordion Summary

**One-liner:** 전략 설정(PaperRulesPage)은 config축 배지+토글만, 전략 운영(TradingRulesPage)은 ACTIVE 필터+run 배지+시작/중지만 노출하고, 백테스트·모의 거래 테이블에 TradeRationaleRow 공유 아코디언을 삽입해 RULE-08/09 UI를 완성.

## Decisions Made

1. **configStatus ?? 'DRAFT' fallback**: 백엔드 V35 마이그레이션 이전에 생성된 레거시 룰은 `configStatus` 필드가 null일 수 있으므로 null coalescing 방어.

2. **parseRationale() 이중 포맷 처리**: 백테스트는 `rationaleJson`이 `{side, exitReason, conditions}` 직접 형태. 모의 이력은 `{price, rsi14, ..., rationale: {...}}` indicatorSnapshot 래퍼 형태. 동일 파서가 양쪽 처리.

3. **expandedId: index vs id**: BacktestTrade는 고유 id가 없어 배열 인덱스 사용. PaperTradeHistoryItem은 id 사용. 두 페이지가 독립적이므로 충돌 없음.

4. **마지막 신호 시각 컬럼 생략**: `PaperLifecycleController.listActive()`가 현재 max(signal_log.ts) JOIN을 반환하지 않아 컬럼 추가 불가. SUMMARY에 명시 후 deferred.

5. **deactivate guard UI**: `runStatus === "RUNNING"` 시 버튼 disabled + tooltip으로 "전략 운영에서 먼저 중지" 안내. 백엔드 ERR_LIFECYCLE_006과 동일 의미.
