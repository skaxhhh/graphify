# Phase 6.5 Plan 02: EvalResult Rich Return Type Summary

**One-liner:** RuleEvaluator에 EvalResult 리치 반환 타입 도입 — 조건별 지표값·passed·ExitReason(TAKE_PROFIT/STOP_LOSS/INDICATOR) 포착, 기존 boolean 시그니처 하위 호환 위임으로 유지.

## What Was Built

### EvalResult.java (신규)

`EvalResult` Java record는 Plan 03(백테스트)와 Plan 04(모의 평가)가 동일 JSON 스키마로 매매 근거를 생성하기 위한 핵심 데이터 소스다.

```java
public record EvalResult(
    boolean triggered,
    List<ConditionResult> conditions,
    ExitReason exitReason,   // null = 진입
    Double exitPct           // null if not TP/SL
) {
    public record ConditionResult(
        String expr,        // "RSI(14) < 30"
        String leftLabel,   // "RSI(14)"
        double leftValue,   // 28.57
        String op,          // "<"
        String rightLabel,  // "30.0"
        double rightValue,  // 30.0
        boolean passed      // true
    ) {}
    public enum ExitReason { TAKE_PROFIT, STOP_LOSS, INDICATOR }
}
```

### RuleEvaluator 확장

신규 메서드 2개:
- `evalEntry(ConditionGroup, double[], Double[], int)` → `EvalResult`
- `evalExit(ExitSpec, double[], Double[], int, double)` → `EvalResult`

내부 private 헬퍼:
- `evaluateGroupRich()` — 조건 목록을 `ConditionResult` 리스트로 변환
- `evaluateConditionRich()` — 단일 조건 평가 (crossAbove/crossBelow 현재 인덱스 값 기록)
- `applyLogic()` — AND/OR 그룹 boolean 합산 (각 ConditionResult.passed 보존)
- `operandLabel()` — Operand → "RSI(14)", "SMA(20)", "30.0" 등 레이블 생성

기존 `entryTriggered`/`exitTriggered`는 각각 `evalEntry().triggered()` / `evalExit().triggered()` 위임으로 변경 — 하위 호환 보존.

### JSON 매핑 의도 (Plan 03/04 참조)

Plan 03(백테스트 TradeDto)와 Plan 04(LiveEvaluationService indicatorSnapshot 병합)에서 사용할 rationale JSON 스키마:


## Decisions Made

1. **operandLabel 포맷**: 상수는 `String.valueOf(Double)` (예: "30.0"), 지표는 `IND(period)` (예: "RSI(14)"). period=0 이면 지표명만 (예: "PRICE").

2. **crossAbove/crossBelow ConditionResult**: leftValue/rightValue = 현재 인덱스 `i` 기준 값. expr = `"EMA(9) crossAbove SMA(20)"`. Pitfall 3 대응.

3. **evalExit 우선순위**: TP → SL → INDICATOR 순서. TP/SL은 `exitPct`(변동률) 반환, INDICATOR는 `exitPct=null`.

4. **기존 boolean 메서드 위임**: `entryTriggered`/`exitTriggered`의 별도 boolean 로직을 제거하고 리치 메서드로 위임. 단일 편집 지점 유지.

5. **TP/SL 시 conditions 빈 List**: TP/SL로 청산될 때는 지표 조건 평가 자체를 건너뛰므로 conditions=`List.of()`. INDICATOR만 조건 리스트 포함.
