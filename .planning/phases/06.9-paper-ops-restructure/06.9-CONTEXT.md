## 문제 정의 (사용자 피드백)

현재 모의(Paper) 콘솔의 메뉴 구성과 흐름이 불친절하다.

1. **모의 대시보드가 무엇을 보여주는지 불명확** — 어떤 테스트/전략을, 어느 기간을 보여주는지 알 수 없다.
   - (확인된 원인) 대시보드는 **계좌 전체 1개의 실시간 스냅샷**이다. 기간 개념이 없다. 기간은 성과 리포트에만 있다.
2. 모의 대시보드 / 모의 거래 이력 / 모의 성과 리포트가 **따로 노는 3개 메뉴**라 맥락이 끊긴다.
3. 동작 모니터링의 시장 상태·스케줄러 정보가 운영 맥락과 분리되어 있고, 최근 신호로그는 노이즈다.
4. 모의 투자 모드인데 공통 메뉴에 토스(실계좌) 설정이 노출된다.

## 현재 데이터 모델 (코드 확인 결과)

| 화면 | 엔드포인트 | 범위 | 기간 | ruleId 보유 |
|------|-----------|------|------|------------|
| 모의 대시보드 | `GET /api/v1/trading/paper/dashboard` | 계좌 전체 1개 (PAPER_LIVE 전략 합산) | 없음(스냅샷) | ✗ |
| 모의 거래 이력 | `GET /api/v1/trading/paper/history` | 계좌 전체 1개 | 전체 누적 | ✗ (`PaperTradeHistoryItem`에 ruleId 없음) |
| 모의 성과 리포트 | `GET /api/v1/trading/paper/report` | 계좌 전체 1개 | `periodFrom~periodTo` 있음 | ✗ |
| 동작 모니터링 | `GET /api/v1/trading/paper/monitor` | 계좌 전체 | 오늘 | 신호로그만 ruleId 보유 |

**핵심 제약:** 거래·포지션·이력·리포트에 **전략 식별자(ruleId)가 붙어있지 않다.** 백엔드는 모의투자를 "전략별/실행별"이 아니라 **하나의 통합 계좌 원장**으로 노출한다. 룰 라이프사이클은 start/stop만 있고 **"실행(run/세션)" 엔티티가 없다.**

→ 따라서 본 phase는 **프론트 전용이 아니다. 백엔드(도메인·DTO·REST) 변경 포함.**

## 확정 결정 (Decisions)

- **D1 — 전략별 분리 + 실행(run) 이력 (백엔드 포함):**
  모의 데이터를 전략별로 구분한다. 한 전략이 **여러 일자에 나눠 실행**될 수 있으므로, 전략별 **실행 이력(run)** 개념을 도입하고, **기간 필터로 특정 실행을 선택하거나 여러 실행을 통합 조회**할 수 있어야 한다.
- **D2 — 통합 메뉴 = master→detail:**
  모의 대시보드 + 거래 이력 + 성과 리포트를 **하나의 메뉴로 통합**한다. 그 메뉴는 먼저 **전략 실행 이력 리스트**(실행 중 + 종료 모두)를 보여주고, 항목을 누르면 **별도 상세 페이지로 이동**해 **3개 탭(대시보드 · 거래 이력 · 성과 리포트)**으로 조회한다. 상세 페이지에 기간 필터 제공.
- **D3 — 동작 모니터링 메뉴 완전 삭제:**
  시장 상태 · 스케줄러 최근 실행 → **전략 운영 화면 상단으로 이동**. 최근 신호로그 · 오늘 거래 → **제거**. `동작 모니터링` 메뉴 자체를 없앤다. (라우트/ModeGuard/페이지 제거)
- **D4 — 토스 설정 PAPER에서 숨김:**
  토스 설정은 **LIVE 모드에서만** 노출. PAPER 모드 공통 메뉴에서 숨긴다. (라우트 유지, 모드 가드/조건부 노출)

- 운영 결과(가칭): **전략 실행 이력 리스트 → 상세(대시보드·거래이력·리포트 3탭 + 기간 필터)**

## 백엔드 작업 범위 (RESEARCH/PLAN에서 구체화)

- **실행(run/세션) 엔티티 도입:** 룰 start→stop 1회 = 1 run(또는 전략별 누적 run 목록). start/end 타임스탬프, 상태(RUNNING/STOPPED), 연결된 ruleId.
- **거래·포지션·에쿼티 태깅:** run(및 ruleId)으로 스코프 가능하게 태그.
- **신규/확장 엔드포인트:** 실행 이력 리스트 조회, run-scoped(또는 rule+기간 scoped) 대시보드/이력/리포트.
- **기간 필터 의미론:** 단일 run 선택 vs 동일 전략의 여러 run 기간 통합 집계 — 둘 다 지원.

## 확정 (wireframe 확인 완료, 2026-06-28)

- **D5 — 단일 계좌 공유 (전략별 서브계좌 아님):** 모든 전략이 하나의 모의 계좌를 공유한다. 거래·포지션·equity는 run(및 ruleId)으로 **태깅**하되, 총자산/가용현금은 **계좌 전체** 값이다.
  → 실행 상세의 "대시보드" 탭은 **해당 run의 기여분**(실현손익·그 run의 포지션·거래수)을 보여주고, 총자산/가용현금은 **계좌 컨텍스트**로 함께 표기. (전략별 자산 완전 분리는 단일 계좌 특성상 "기여분" 방식으로 근사)
- **D6 — run 정의:** 룰 start→stop 1회 = 1 run. 같은 전략 재실행 = 새 run(회차). (wireframe 안 확정)
- **D7 — 기간 필터 2-mode:** 상세 페이지에서 "이 실행만 ↔ 전략 전체 통합"(동일 전략의 여러 run 합산) + 기간 선택. (wireframe 안 확정)
- **D8 — 통합 메뉴 명칭:** "운영 결과 > 실행 이력" (`/trading/paper/runs`, 상세 `/trading/paper/runs/:runId`).
- **실행 이력 리스트 컬럼:** 전략명·상태(실행중/종료)·실행 기간·유니버스·수익률·실현손익·거래수·최종 자산. (wireframe 안 확정)
