# Phase 6.9 Plan 02: run_id Stamping + Aggregation Services Summary

**One-liner:** PaperExecutor now stamps paper_trades.run_id via resolveActiveRunId() at each BUY/SELL, and two new services aggregate run-scoped contribution (realizedPnl, tradeCount, open positions) and RULE_AGGREGATE period-filter stats.

## What Was Built

**Task 1 — PaperExecutor run_id Stamping (UIX-02-BE-3, BE-4)**

`PaperExecutor.java` gains `PaperRunRepository runRepo` as 7th constructor param. A private `resolveActiveRunId(Long ruleId)` helper calls `runRepo.findFirstByRuleIdAndStatus(ruleId, "RUNNING").map(PaperRun::getId).orElse(null)`. Both edit points are updated:
- `executeBuy()`: `Long runId = resolveActiveRunId(rule.getId())` → `new PaperTrade(accountId, ruleId, runId, symbol, "BUY", qty, price, null, ts)`
- `executeSell()`: same resolver → `new PaperTrade(accountId, ruleId, runId, symbol, "SELL", qty, price, pnl, ts)`

If no RUNNING run exists, `runId=null` is stamped — backward compat for legacy trades (D7 go-forward-only policy).

`PaperExecutorRunTagTest` validates all 3 cases with `ArgumentCaptor<PaperTrade>`: BUY stamps runId, SELL stamps runId + pnl, no-run yields null.

`PaperExecutorTest` updated to 7-arg constructor + lenient runRepo default stub.

**Task 2 — PaperTradeRepository queries + PaperRunListService + PaperRunContributionService (UIX-02-BE-7)**

`PaperTradeRepository` gains:
- `findByRunIdOrderByTradedAtDesc(Long runId)` — run-scoped, excludes NULL rows (Spring Data WHERE run_id = :runId)
- `findByRuleIdAndTradedAtBetweenOrderByTradedAtDesc(Long ruleId, Instant from, Instant to)` — RULE_AGGREGATE, includes NULL run_id legacy trades

`PaperRunListService.listRuns(userId)` returns `List<RunListItem>` with all aggregated columns:
- `realizedPnl` = SUM(pnl WHERE SELL)
- `returnPct` = realizedPnl / SUM(qty×price WHERE BUY) × 100 (denominator = actual invested capital)
- `tradeCount` = COUNT(BUY) (진입 횟수)
- `finalEquity` = RUNNING→latest snapshot equity, STOPPED→last snapshot before ended_at
- `runIndex` = 1-based position within same ruleId group sorted by startedAt ASC (real-time, not stored)
- `universe` = parsed from `paper_runs.universe_snapshot` JSON

`PaperRunContributionService` provides two modes:
- `dashboardContribution(userId, runId)` → `RunDashboardContribution` with totalEquity/availableCash (account-wide, D5), realizedPnl/tradeCount (run-scoped), open positions derived from BUY-without-SELL symbols → positionRepo lookup → mark-to-market via intradayRepo
- `aggregate(ruleId, from, to)` → `RuleAggregateResult` (RULE_AGGREGATE 2-mode, includes NULL run_id trades)

`PaperRunContributionServiceTest` covers: RULE_AGGREGATE with NULL run_id legacy trades, single-run closed position, single-run open position derivation.

- **Issue:** `when(runRepo.findFirstByRuleIdAndStatus(...)).thenReturn(Optional.of(mockRunWithId(RUN_ID)))` called `mock(PaperRun.class)` inside the outer `when()` chain, leaving Mockito's stubbing state unfinished

**2. [Rule 1 - Bug] TradingRule first constructor arg is userId not id — rule.getId() returns null in unit tests**

- **Files modified:** `PaperExecutorRunTagTest.java`
