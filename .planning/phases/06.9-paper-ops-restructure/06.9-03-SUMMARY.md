# Phase 6.9 Plan 03: Run-Scoped REST API (Wave 3) Summary

**One-liner:** Two new controllers expose GET /runs (list with 8 columns) and GET /runs/{runId}/dashboard|history|report (2-mode SINGLE_RUN vs RULE_AGGREGATE), completing the backend contract for Wave 4/5 frontend.

## What Was Built

**Task 1 — RunSummaryDto + RunDashboardDto + PaperRunListController + Test (UIX-02-BE-5)**

`RunSummaryDto` record (12 fields): `runId, ruleId, ruleName, runIndex, status, startedAt, endedAt, universe, realizedPnl, returnPct, tradeCount, finalEquity`. Captures all 8+ columns in RESEARCH endpoint spec.

`RunDashboardDto` record (10 fields): D5 split — `totalEquity/availableCash` (account-wide) + `realizedPnl/unrealizedPnl/tradeCount/positions` (run-scoped).

`PaperRunListController` at `GET /api/v1/trading/paper/runs`:
- Calls `listService.listRuns(userId)` → `List<RunListItem>`
- Inline stream mapping to `List<RunSummaryDto>`
- Returns `ApiResponse.ok(runs)` following existing controller convention

`PaperRunListControllerTest` (4 tests, UIX-02-BE-5):
- `listRuns_stoppedRun_computesAllColumns`: realizedPnl, returnPct (BUY invested cost denominator), tradeCount (BUY only), finalEquity (snapshot before endedAt)
- `listRuns_runningAndStopped_correctRunIndex`: RUNNING=2회차, STOPPED=1회차 mixed list
- `listRuns_noRuns_returnsEmpty`: NPE-safe empty path
- `listRuns_stoppedRun_tradeCountIsBuyOnly`: tradeCount=BUY count, realizedPnl=SELL pnl sum

**Task 2 — PaperRunDetailController + Service Overloads + Dashboard Test (UIX-02-BE-6)**

`PaperHistoryService` — `buildHistoryItems(List<PaperTrade>)` private helper extracted from `getHistory()`. Two new public methods share the helper:
- `getHistoryByRun(userId, runId)` → `findByRunIdOrderByTradedAtDesc` (SINGLE_RUN mode)
- `getHistoryByRuleAndPeriod(userId, ruleId, from, to)` → `findByRuleIdAndTradedAtBetweenOrderByTradedAtDesc` (RULE_AGGREGATE mode, includes legacy NULL run_id)

`PaperReportService` — `buildReportForPeriod(accountId, from, to)` private helper for period-scoped equity curves. Two new public methods:
- `getReportByRun(userId, startedAt, endedAt)` → `buildReportForPeriod(accountId, startedAt, endedAt ?? now)`
- `getReportByPeriod(userId, from, to)` → `buildReportForPeriod(accountId, from, to)`
- `computeTradeStatsForPeriod(accountId, from, to)` filters SELL trades to the period

`PaperRunContributionService` — `IllegalArgumentException` → `GraphifyException("ERR_PAPER_RUN_001", HttpStatus.NOT_FOUND)` for proper HTTP 404 response.

`PaperRunDetailController` at `GET /api/v1/trading/paper/runs/{runId}`:
- `GET /dashboard` → `contributionService.dashboardContribution(userId, runId)` → `RunDashboardDto` (adds runIndex + ruleName lookup)
- `GET /history?mode=RULE_AGGREGATE&from=...&to=...` → 2-mode dispatch
- `GET /report?mode=RULE_AGGREGATE&from=...&to=...` → 2-mode dispatch
- `computeRunIndex(run)`: `findByRuleIdOrderByStartedAtDesc` sorted ASC → 1-based position
- `findRunOrThrow(runId)`: `GraphifyException NOT_FOUND` if runId not in DB

`PaperRunDashboardControllerTest` (5 tests, UIX-02-BE-6):
- `dashboardContribution_returnsRunScopedPnlAndAccountWideEquity`: totalEquity = cash + ALL positions (D5), realizedPnl = run SELL pnl
- `dashboardContribution_openPositions_derivedFromRunTrades`: BUY-without-SELL → open position with unrealizedPnl
- `dashboardContribution_runNotFound_throwsNotFound`: GraphifyException ERR_PAPER_RUN_001 / 404
- `getHistoryByRun_returnRunScopedTrades`: run_id-scoped history with rationale JOIN
- `getHistoryByRuleAndPeriod_returnsRuleAndPeriodScopedTrades`: RULE_AGGREGATE includes NULL run_id legacy trades

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Integer literals don't auto-box to `Double` param type in test helper**
- **Found during:** Task 1 test compilation
- **Issue:** `makeTrade(..., 200_000)` passed `int` to `Double pnl` param; Java doesn't auto-widen int→Double (boxed)
- **Fix:** Changed all pnl literals to `200_000.0`, `300_000.0`, `150_000.0`
- **Files modified:** `PaperRunListControllerTest.java`

**2. [Rule 1 - Bug] UnnecessaryStubbingException on `account.getCash()` when snapshots present**
- **Found during:** Task 1 tests — `listRuns_stoppedRun_computesAllColumns`, `listRuns_runningAndStopped_correctRunIndex`
- **Issue:** Mockito strict mode rejects stubs that are never called. `account.getCash()` is only the fallback in `resolveCurrentEquity` when no snapshots exist; when snapshots ARE present the snapshot equity is used instead
- **Fix:** Changed two `when(account.getCash())` calls to `lenient().when(account.getCash())`
- **Files modified:** `PaperRunListControllerTest.java`

**3. [Rule 1 - Bug] UnnecessaryStubbingException on `positionRepo.findByAccountIdAndSymbol` + `symbolNameService.resolve` when BUY+SELL cancel out**
- **Found during:** Task 2 tests — `dashboardContribution_returnsRunScopedPnlAndAccountWideEquity`
- **Issue:** Test had both BUY and SELL for same symbol; openSymbols = empty after subtraction; per-symbol position and name lookups were never reached
- **Fix:** Changed those stubs to `lenient().when()`
- **Files modified:** `PaperRunDashboardControllerTest.java`
