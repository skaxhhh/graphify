# Phase 6.9 Plan 01: V37 Migration + PaperRun Lifecycle Summary

**One-liner:** V37 Flyway migration creates paper_runs table + adds paper_trades.run_id; PaperLifecycleService.start/stop now create and close PaperRun rows within the same @Transactional boundary.

## What Was Built

**Task 1 — V37 Migration + PaperRun Entity + Repository**

`V37__paper_runs.sql` creates the `paper_runs` table with columns: `id BIGSERIAL`, `rule_id BIGINT NOT NULL`, `user_id BIGINT NOT NULL`, `started_at TIMESTAMPTZ`, `ended_at TIMESTAMPTZ` (NULL = in-progress), `status VARCHAR(8)` (RUNNING/STOPPED check constraint), `universe_snapshot TEXT`. Three indexes: `(rule_id)`, `(user_id, started_at DESC)`, `(status)`. Additionally alters `paper_trades` to add `run_id BIGINT` (nullable) with an index, and backfills synthetic "RUNNING" rows from `trading_rules.run_status='RUNNING'` for orphaned in-flight rules at migration time.

`PaperRun.java` is a standard JPA entity with `stop(Instant endedAt)` lifecycle method and a public 4-arg constructor that initializes status to "RUNNING". `PaperRunRepository` extends `JpaRepository` with three Spring Data methods: `findByUserIdOrderByStartedAtDesc`, `findByRuleIdOrderByStartedAtDesc`, `findFirstByRuleIdAndStatus`.

**Task 2 — PaperTrade runId + start/stop lifecycle + Tests (TDD)**

`PaperTrade.java` gains `run_id` nullable column, a new 9-arg primary constructor, and the existing 8-arg constructor delegates to it with `runId=null` for backward compatibility (Wave 2 PaperExecutor edits will upgrade these call sites).

`PaperLifecycleService.java` gains `PaperRunRepository runRepo` as a 7th constructor dependency. `start()` now calls `runRepo.save(new PaperRun(saved.getId(), userId, Instant.now(), universeJson))` after assigning symbols within the same `@Transactional` block (Pitfall 5 prevention). `stop()` calls `runRepo.findFirstByRuleIdAndStatus(saved.getId(), "RUNNING").ifPresent(run -> { run.stop(Instant.now()); runRepo.save(run); })` — ifPresent means no-op if no matching run exists.
