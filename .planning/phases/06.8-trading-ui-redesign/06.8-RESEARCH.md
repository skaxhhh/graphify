## Summary

Phase 6.8는 `/trading/**` 15개 화면을 Binance 다크 테마로 리스킨하고 IA를 재배치하는 순수 프론트엔드 작업이다. Wave 0 토대(trade-* 토큰 17종 + Inter/JetBrains Mono 웹폰트)는 이미 완료되어 `tailwind.config.js`와 `index.html`에 반영되어 있다.

기존 모든 트레이딩 페이지는 `bg-gray-950/900/800`, `text-gray-*`, `emerald-*`, `red-*`, `border-white/*` 등 cream 시스템의 다크 변형 클래스를 직접 사용한다. 이 클래스들은 trade 토큰으로 **전면 교체**해야 한다(덧씌우기 금지 — Tailwind specificity 충돌). 총 교체 대상은 트레이딩 페이지 206줄 + 레이아웃 12줄 + trading 컴포넌트(PaperTradingToggle, TradeRationaleRow 등) 다수.

차트 컴포넌트(`EquityCurveChart`, `CandleChart`)는 recharts/lightweight-charts 내부 SVG/Canvas에 inline hex를 사용하므로 D2에 따라 내부를 건드리지 않는다. 대신 외부 래핑 컨테이너(currently `bg-gray-900/50 border-white/10`)를 trade surface 토큰으로 교체한다. `CandleSection`은 `EmptyState`/`ErrorBanner`/`SkeletonBlock`에 `tone="dark"`를 전달하나, 이 컴포넌트들의 "dark" 톤은 cream 시스템 gray 팔레트라 trade bg 위에서 이질감이 생긴다 — Wave 0에서 `TradePageState` 프리미티브 신설로 해결한다.

**Primary recommendation:** Wave 0에서 `components/trading/ui/` 프리미티브를 먼저 확립하고, 이후 각 wave에서 해당 프리미티브만 사용하면 토큰 일탈 없이 기계적으로 진행 가능하다.

### Pattern 1: trade 전용 프리미티브 — components/trading/ui/

**What:** shared/ 컴포넌트를 대체하는 trade 다크 테마 전용 버전. shared/를 직접 수정하지 않는다(D6).

**When to use:** /trading/** 안의 모든 UI 요소.

**Example — TradeButton:**
```typescript
// components/trading/ui/TradeButton.tsx
// Source: DESIGN-binance.md button-primary / button-secondary-on-dark / button-primary-disabled
interface TradeButtonProps {
  variant?: "primary" | "secondary" | "danger" | "ghost";
  size?: "sm" | "md";
  disabled?: boolean;
  loading?: boolean;
  onClick?: () => void;
  children: React.ReactNode;
}

export function TradeButton({ variant = "primary", size = "md", disabled, loading, onClick, children }: TradeButtonProps) {
  const base = "inline-flex items-center justify-center rounded font-trade-sans font-semibold transition-colors focus:outline-none focus:ring-2 focus:ring-trade-info";
  const sizes = { sm: "h-7 px-3 text-xs", md: "h-10 px-5 text-sm" };
  const variants = {
    primary:   "bg-trade-primary text-trade-ink hover:bg-trade-primary-active disabled:bg-trade-primary-disabled disabled:text-trade-muted",
    secondary: "bg-trade-surface text-trade-body border border-trade-hairline hover:bg-trade-elevated",
    danger:    "bg-trade-down text-trade-on-dark hover:opacity-90",
    ghost:     "bg-transparent text-trade-body hover:bg-trade-elevated",
  };
  return (
    <button type="button" disabled={disabled || loading} onClick={onClick}
      className={`${base} ${sizes[size]} ${variants[variant]} disabled:cursor-not-allowed`}>
      {loading ? "..." : children}
    </button>
  );
}
```

**Example — TradeStatCard:**

**Example — TradePageState (loading/empty/error):**
```typescript
// components/trading/ui/TradePageState.tsx
// Replaces EmptyState tone="dark"/ErrorBanner tone="dark"/SkeletonBlock tone="dark"
// Uses trade tokens instead of cream-system gray palette
type TradePageStateVariant = "loading" | "empty" | "error";

interface TradePageStateProps {
  variant: TradePageStateVariant;
  title?: string;
  message?: string;
  onRetry?: () => void;
  className?: string;
}

export function TradePageState({ variant, title, message, onRetry, className = "" }: TradePageStateProps) {
  if (variant === "loading") {
    return (
      <div className={`animate-pulse rounded-lg bg-trade-surface border border-trade-hairline ${className}`} />
    );
  }
  return (
    <div className={`flex min-h-[200px] flex-col items-center justify-center rounded-lg border border-trade-hairline bg-trade-surface px-6 py-10 text-center ${className}`}>
      <p className="text-sm font-medium text-trade-body">{title}</p>
      {message && <p className="mt-1 text-xs text-trade-muted">{message}</p>}
      {onRetry && (
        <button type="button" onClick={onRetry}
          className="mt-4 text-xs text-trade-primary hover:text-trade-primary-active underline">
          재시도
        </button>
      )}
    </div>
  );
}
```

### Anti-Patterns to Avoid

- **클래스 중복 적용:** `bg-gray-900 bg-trade-surface` 처럼 old 클래스를 남기면 예측 불가 결과 발생. 반드시 old 클래스 제거 후 trade 토큰 교체.
- **hex 직접 사용:** `style={{ color: '#fcd535' }}` 금지. 반드시 `text-trade-primary` 사용.
- **shared/ 컴포넌트 수정:** `EmptyState`/`PrimaryButton` 등을 trade 테마로 수정 금지 — cream 앱 회귀 유발. 대신 `components/trading/ui/` 신설.
- **trade-up/down을 배경 fill로 사용:** `bg-trade-up` (카드 배경) 금지 — 텍스트 색(`text-trade-up`) 또는 소프트 배경(`bg-trade-up-soft`)만 허용.
- **font-trade-mono 전역 적용:** 레이아웃 루트에 `font-trade-mono` 적용 금지. 숫자/가격 요소에만 인라인 적용.

## Common Pitfalls

### Pitfall 1: 토큰 중복 적용 (가장 흔한 실수)
**What goes wrong:** 기존 `bg-gray-900` 위에 `bg-trade-surface`를 추가 → Tailwind JIT가 두 클래스 모두 생성, CSS specificity가 동일하면 마지막 정의 클래스 승리 (예측 불가).
**Why it happens:** 리스킨 시 기존 클래스를 지우지 않고 trade 클래스를 append.
**How to avoid:** Find → Replace 방식으로 old class를 완전히 제거 후 교체. grep으로 잔존 old 클래스 확인.
**Warning signs:** `bg-gray-*` 클래스가 `/trading/` 파일에 남아 있으면 불합격.

### Pitfall 2: EquityCurveChart 내부 색상 변경 시도
**What goes wrong:** D2를 위반해 recharts `stroke="#10b981"`을 `text-trade-up` hex로 변경 → trade-up(`#0ecb81`)과 미묘하게 달라 시각적 불일치.
**How to avoid:** EquityCurveChart.tsx와 CandleChart.tsx는 phase 6.8 범위에서 diff가 0이어야 한다.

### Pitfall 3: ModeGuard LIVE→PAPER 변경 후 nav 미동기화
**What goes wrong:** router의 monitor ModeGuard는 PAPER로 바꿨는데 TradingLayout의 `liveItems`에 monitor가 남아 있음 → PAPER 모드에서 liveItems가 표시되지 않으니 monitor 링크 사라짐.
**How to avoid:** router 변경 + TradingLayout nav 배열 변경을 동일 커밋에서 수행.

### Pitfall 4: EmptyState/ErrorBanner tone="dark" 잔존
**What goes wrong:** CandleSection이 기존 `EmptyState tone="dark"`를 유지 → `bg-gray-900/50 border-white/10` 사용 → trade-bg(`#0b0e11`) 위에서 시각적 이질감.
**How to avoid:** CandleSection의 4개 상태를 `TradePageState`로 교체. (CandleChart canvas 자체는 건드리지 않음.)

### Pitfall 5: CompanyPickerModal cream 컴포넌트 노출
**What goes wrong:** 종목 선택 모달이 열릴 때 `PrimaryButton`(cream yellow), `TextField`(cream light bg)가 trade 다크 배경 위에 렌더링 → 심각한 테마 불일치.
**How to avoid:** Wave 1에서 CompanyPickerModal 리스킨 또는 TradingCompanyPickerModal 신설 우선 처리.

### Pitfall 6: font-trade-mono 범위 오적용
**What goes wrong:** `font-trade-mono`를 TradingLayout 루트에 적용 → 모든 버튼·레이블·헤딩이 모노스페이스 → "trading-platform character" 파괴.
**How to avoid:** `font-trade-sans`만 루트에. `font-trade-mono`는 가격/지표값/코드 요소에만 명시적으로 적용.

### Pitfall 7: PaperTradingToggle LIVE 전환 모달 (LIVE 위험 CTA)
**What goes wrong:** 기존 `bg-red-600` (CTA) → trade-down 교체가 자연스러우나, Binance 위험 CTA는 오히려 `bg-trade-primary`(yellow)를 사용하는 경우가 많음. D8 wireframe 확인 필요.
**CONTEXT.md D8:** "옐로우 확인 CTA" 명시 → `bg-trade-primary text-trade-ink` 사용.
**How to avoid:** LIVE 전환 확인 버튼: `bg-trade-primary text-trade-ink` (yellow, 위험 강조용). 취소: `bg-trade-surface border-trade-hairline`.

### Pitfall 8: Playwright 기존 e2e 문자열 의존성
**What goes wrong:** `candle-chart.spec.ts`가 DOM 텍스트("캔들 데이터를 불러오지 못했습니다." 등)로 상태를 판별. 리스킨 중 한국어 텍스트 변경 시 테스트 실패.
**How to avoid:** 리스킨 시 기존 텍스트 문자열 변경 금지. TradePageState의 message prop에 동일 문자열 전달.
