# Phase 6.8 — Trading 콘솔 UI 개편 (Binance 테마 리스킨 + IA 재배치)

**Type**: UI redesign (frontend-only, 기능 동작 불변)
**Depends on**: Phase 6.7 (현재 구현 전부)
**Inputs**: `wireframe.html`(화면 SoT) · `DESIGN-binance.md`(토큰 SoT) · `PRODUCT_MANIFEST.md`(기능 분해) · `plan.md`(요구사항 인벤토리)
**Status**: DISCUSS (결정 확정 → plan-phase 진행 예정)

---

## Goal

현재 `/trading/**` 화면 전체를 `wireframe.html` 기준 **Binance 다크 트레이딩 테마**로 리스킨하고 IA를 재배치한다. **백엔드·API·비즈니스 로직·차트 엔진은 변경하지 않는다** — 레이아웃·테마·메뉴 구조·컴포넌트 정렬만 바꾼다. 빅뱅이지만 화면 단위 상세 기획(매핑표 + 상태 명세 + 시각 검증 게이트)으로 통제한다.

## 비목표 (Out of scope, 이번 phase)
- 백엔드/REST/DTO 변경, 차트 라이브러리 교체, 신규 기능(Phase 7 TradingView·Phase 8 LIVE 실데이터)의 **실구현**. → 미래 슬롯은 **시각 placeholder만** 배치.
- DDS Agent 실연동(현재 목업 유지, 리스킨만).

## 결정 사항 (DISCUSS — 확정 후 plan-phase)

`[확정]` = 사용자/wireframe로 이미 확정, `[제안]` = 권고안(확인 필요).

- **D1 [확정] 테마 범위**: `trade-*` 토큰은 `/trading/**` 전용. 앱 본체(cream) 불변. (구현 완료)
- **D2 [확정] 차트 재사용**: `EquityCurveChart`·`CandleSection`(recharts/lightweight-charts) **현재 구현 그대로**. 다크 surface로 감싸고 색만 trade 토큰 정렬. 차트 내부 로직·라이브러리 교체 없음.
- **D3 [확정] 동작 모니터링 귀속(A3)**: LIVE 메뉴 → **PAPER 그룹으로 이동**(데이터가 paper/monitor). 라우트 `/trading/monitor` 유지하되 PAPER 모드에서 노출. LIVE 모니터는 Phase 8에서 실데이터로 별도 구현.
- **D4 [확정] 메뉴 명칭 통일**: wireframe대로 **"전략 설정 / 전략 운영"** 채택. 레거시 라벨("모의 룰 설정/룰 라이프사이클/현재 룰") 폐기.
- **D5 [확정] DDS Agent**: 이번 phase는 **리스킨만**(목업 유지). 실연동은 별도(Should/Later, API 스펙 미정).
- **D6 [확정] 컴포넌트 전략**: **`components/trading/ui/` 아래 trade 전용 프리미티브 신설**(TradeButton·TradeBadge·TradeCard·TradeTable·TradePageState 등). 기존 `shared/`(cream) 비파괴.
- **D7 [확정] 미래 슬롯**: LIVE 화면·Phase 7/8은 **비활성 placeholder만** 배치(coming-soon/준비 중), 실구현 X.
- **D8 [확정] 모드 토글 위치**: 사이드바 하단 세그먼트(모의/실거래) 유지 + 상단 모드 인디케이터 추가. LIVE 전환 위험 모달 유지.

## 수용 기준 (Success Criteria)
1. `/trading/**` 전 화면이 trade 토큰·폰트만 사용(하드코딩 hex 0건), wireframe 레이아웃과 구조 일치.
2. 모든 기존 기능(룰 CRUD·2축 상태·백테스트·폴링·근거·모달·모드 가드) 동작 불변.
3. 차트는 기존 컴포넌트 재사용, 다크 테마로 정렬.
4. 빈/로딩/에러/성공/disabled 상태가 wireframe 주석대로 구현.
5. cream 앱 본체·기존 라우트 회귀 없음.
6. LIVE·Phase7/8은 비활성 placeholder로 존재(실구현 아님).
