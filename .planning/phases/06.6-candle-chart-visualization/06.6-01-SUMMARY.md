## What Was Built

- **MarketBarIntraday.java**: Added `getOpen()`, `getHigh()`, `getLow()` public getters alongside existing `getClose()`/`getVolume()`. Fields existed but had no public accessor ‚Äî Wave 0 gap closed.

- **CandleBarDto.java**: New `record CandleBarDto(long time, double open, double high, double low, double close, long volume)` in `com.graphify.trading.backtest.dto`. Static factory `CandleBarDto.from(MarketBarIntraday)` converts to lightweight-charts shape: `time = ts.getEpochSecond()` (seconds, not millis), OHLC fields fall back to `close` when null.

- **BacktestController.java**: Injected `MarketBarIntradayRepository`. Added `GET /bars` endpoint accepting `?symbol=&date=` (ISO date). Builds KST day Instant range (`atStartOfDay` ‚Üí `23:59:59`), calls `findBySymbolAndRange`, streams to `CandleBarDto`, returns `ApiResponse.ok(dtos)`. Inherits session auth from existing `/api/v1/trading/**` security config ‚Äî no SecurityConfig change.

- **BarQueryTest.java**: `@DataJpaTest` slice (Flyway disabled, ddl-auto=create-drop, H2 in PostgreSQL mode). Five tests:
  - `candleBarDto_from_maps_epoch_seconds` ‚Äî API-2: time < 1e11 (seconds, not millis)
  - `candleBarDto_from_maps_ohlcv_correctly` ‚Äî OHLCV fields mapped correctly
  - `candleBarDto_from_null_ohlc_falls_back_to_close` ‚Äî null OHLC fallback
  - `findBySymbolAndRange_returns_only_requested_kst_day` ‚Äî API-1: KST day filtering + ascending order
  - `findBySymbolAndRange_bars_map_to_epoch_seconds` ‚Äî API-2 via full mapping chain
