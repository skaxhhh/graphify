## What Was Built

**lightweight-charts@5.2.0** installed (Apache 2.0, tree-shakeable).

**CandleChart.tsx** — fully prop-driven React component:
- Props: `bars`, `trades`, `filterDate`, `indicatorLines`, `highlightTime`
- v5 API: `addSeries(CandlestickSeries)` + `addSeries(HistogramSeries)` + `addSeries(LineSeries)` (NOT deprecated v4 `addCandlestickSeries`)
- `createSeriesMarkers(candleSeries, markers)` (NOT deprecated `series.setMarkers`)
- Volume overlay on separate `priceScaleId: ""` with `scaleMargins { top: 0.8, bottom: 0 }`
- Dark theme: `ColorType.Solid` gray-900 bg, gray-400 text, `rgba(255,255,255,0.05)` grid
- `autoSize: true` (ResizeObserver) + `w-full` + explicit `height: 400` (RESEARCH Pitfall 5)
- Cleanup: `return () => chart.remove()` (StrictMode safe, RESEARCH Pitfall 6)
- Hex → Tailwind token mapping documented in file header (CLAUDE.md compliance)

**candleIndicators.ts** — four pure TS functions (no React):
- `extractIndicators(def)` — scans RuleDefinition for SMA/EMA operands with period; dedupes by `indicator-period` key; excludes RSI/VOLUME (non-price-scale)
- `computeSMA(bars, period)` — sliding-window mean of close, matches Indicators.java
- `computeEMA(bars, period)` — standard EMA seeded with SMA, k = 2/(period+1), matches Indicators.java
- `tradesToMarkers(trades, filterDate)` — BUY = emerald-500 arrowUp belowBar "B", SELL = red-500 arrowDown aboveBar "S"

**types/trading.ts** — added `CandleBar` interface (time/open/high/low/close/volume).

**lib/ruleApi.ts** — added `fetchBars(symbol, date)` → `GET /api/v1/trading/paper/backtest/bars`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] TypeScript noUncheckedIndexedAccess on array element access**
- **Found during:** Task 1 build verification
- **Issue:** `bars[i].close` and `bars[i-period].close` were `T | undefined` under `noUncheckedIndexedAccess: true` (tsconfig.json)
- **Fix:** Added `!` non-null assertions (`bars[i]!.close`) with inline comments explaining bounds are guaranteed by loop condition and early return guards
- **Files modified:** `frontend/src/components/backtest/candleIndicators.ts`
- **Commit:** 75beb34 (included in Task 1 commit)

**2. [Rule 1 - Bug] ColorType string literal rejected by lightweight-charts v5 TypeScript types**
- **Found during:** Task 2 build verification
- **Issue:** `{ type: "solid" as const }` failed with `Type '"solid"' is not assignable to type 'ColorType | undefined'` — the library uses an enum, not a string union
- **Fix:** Import `ColorType` from `lightweight-charts` and use `ColorType.Solid`
- **Files modified:** `frontend/src/components/backtest/CandleChart.tsx`
- **Commit:** 0f55be9 (included in Task 2 commit)
