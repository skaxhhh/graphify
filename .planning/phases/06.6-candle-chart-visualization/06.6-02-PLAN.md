## UI/UX

This plan builds the chart component in isolation; full page wiring + 4-state shell is Plan 03. The component's own visual contract:

**Screen inventory (component scope):**
- CandleChart canvas (success state only — empty/loading/error are owned by the parent section in Plan 03). When `bars.length === 0` the component renders nothing (returns early); the parent decides what to show instead.

**shared/ reuse mapping:**
- No shared/ primitives inside CandleChart — it is a canvas wrapper (lightweight-charts owns rendering). EmptyState/ErrorBanner/SkeletonBlock are applied by the parent section in Plan 03.
- New component justified: there is no existing candlestick primitive; EquityCurveChart is recharts (static) and not reusable for interactive candles.

**Tokens / theme:**
- Dark theme consistent with both pages (`bg-gray-900/50`, `border-white/10`). lightweight-charts requires literal color strings; use the standard Tailwind palette hex documented in the component (emerald-500/red-500/amber-500/gray-900/gray-400). Do NOT use custom cream/charcoal tokens (light theme — RESEARCH Pitfall 7).

**Responsive / keyboard / focus:**
- `autoSize: true` (ResizeObserver) + `w-full` container with explicit height → responsive width. Zoom/pan via lightweight-charts default mouse/touch interactions (locked decision §5). No keyboard interaction expected on the canvas itself; keyboard selection happens via the trade table rows in Plan 03.
