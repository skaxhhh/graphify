### Human Verification Required

#### 1. Candle marker shape and color (SC-3 pixel-level)

**Test:** Run a backtest with trades. Observe the candle chart section.
**Expected:** BUY trades show an emerald (#10b981) upward arrow below the candle bar; SELL trades show a red (#ef4444) downward arrow above the candle bar.
**Why human:** lightweight-charts renders markers to a canvas element — shape and color cannot be asserted via DOM queries.

#### 2. SMA/EMA indicator line overlay (backtest page only)

**Test:** Run a backtest with a rule that references SMA or EMA operands (e.g., SMA(5) > SMA(20)). Observe the candle chart.
**Expected:** Amber (#f59e0b) line(s) overlaid on the candle chart matching the indicator period(s) from the rule definition.
**Why human:** Indicator line rendering is canvas-only; cannot verify color/data visually programmatically.

#### 3. All 4 states are dark-themed (no light cream boxes)

**Test:** Force each of the 4 states:
- No selection: result with 0 trades
- Loading: throttle network during bars fetch
- Error: return 500 from `/bars` endpoint; confirm "재시도" button appears
- Empty bars: bars endpoint returns `[]` for the selected date
**Expected:** All states render dark (gray-900/50 background, gray-500 text, red-500/10 error). No cream or bright-white boxes.
**Why human:** Tailwind class rendering and visual contrast require browser inspection.

#### 4. Row click scrolls to clicked marker and emphasizes it

**Test:** With multiple trades on the same day, click a later trade row.
**Expected:** The clicked trade's marker is larger (size: 2) and the chart time scale scrolls so the marker is visible.
**Why human:** `chart.timeScale().scrollToRealTime()` and marker size are canvas behaviors invisible to the DOM.

#### 5. Zoom/pan works on the chart

**Test:** Scroll/drag on the candle chart.
**Expected:** Chart zooms and pans smoothly (lightweight-charts default interaction).
**Why human:** Mouse interaction on canvas cannot be reliably tested via Playwright at pixel level.
