## Summary

Phase 6.6 adds a lightweight-charts v5 candlestick chart to PaperBacktestPage (below the existing recharts equity curve) and PaperHistoryPage. A single `CandleChart` React component wraps the lightweight-charts lifecycle and is parameterised by `(trades, selectedSymbol, selectedDate, ruleDef)`. When a trade row is clicked, the component fires a GET request to a new backend endpoint that returns the full session 5-minute OHLCV bars for that (symbol, date) pair from `market_bars_intraday`. The bars are rendered as candlesticks, trades are overlaid as `createSeriesMarkers` primitives (emerald arrowUp below / red arrowDown above), volume is shown as a histogram overlay using `priceScaleId: ''` + `scaleMargins`, and rule-defined indicators (SMA/EMA periods extracted from the rule definition) are rendered as lightweight-charts LineSeries overlays computed on the frontend from the returned close prices.

The key architectural insight is that `loadBars()` in `IntradayBacktestEngine` already implements the correct DB-first/Yahoo-fallback logic via `MarketBarIntradayRepository.findBySymbolAndRange(symbol, fromInstant, toInstant)`. The new endpoint can expose this same query through a dedicated `@GetMapping` on `BacktestController` (or a new `MarketBarsController`), returning a typed DTO list wrapped in `ApiResponse<List<CandleBarDto>>`.

**Primary recommendation:** Install `lightweight-charts@^5.2.0`, build `CandleChart.tsx` with `useEffect` chart lifecycle + `createSeriesMarkers` primitive, compute indicator lines client-side from bars data, and add a single `GET /api/v1/trading/paper/backtest/bars` endpoint reusing `MarketBarIntradayRepository.findBySymbolAndRange`.

### Pattern 1: lightweight-charts React lifecycle (createChart / dispose)

**What:** Use `useRef` for DOM container + chart instance references. Create chart in `useEffect`, return cleanup that calls `chart.remove()`. Re-create when `(symbol, date)` change by including them in the dependency array — OR use a stable chart instance with `series.setData()` on data change.

**When to use:** Always — this is the official v5 React pattern.

**Example:**

```typescript
// Source: https://tradingview.github.io/lightweight-charts/tutorials/react/simple
import { createChart, CandlestickSeries, HistogramSeries, LineSeries } from 'lightweight-charts';
import { createSeriesMarkers } from 'lightweight-charts';
import { useEffect, useRef } from 'react';

export function CandleChart({ bars, trades, indicatorLines }: CandleChartProps) {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!containerRef.current || bars.length === 0) return;

    const chart = createChart(containerRef.current, {
      layout: {
        background: { type: 'solid', color: '#111827' }, // gray-900
        textColor: '#9ca3af',                            // gray-400
      },
      grid: {
        vertLines: { color: 'rgba(255,255,255,0.05)' },
        horzLines: { color: 'rgba(255,255,255,0.05)' },
      },
      autoSize: true,
      height: 400,
    });

    // Candlestick series (v5: addSeries(CandlestickSeries, options))
    const candleSeries = chart.addSeries(CandlestickSeries, {
      upColor: '#10b981',      // emerald-500
      downColor: '#ef4444',    // red-500
      borderVisible: false,
      wickUpColor: '#10b981',
      wickDownColor: '#ef4444',
    });
    candleSeries.setData(bars);

    // Volume histogram overlay (priceScaleId '' = no separate scale)
    const volumeSeries = chart.addSeries(HistogramSeries, {
      color: '#374151',        // gray-700
      priceFormat: { type: 'volume' },
      priceScaleId: '',
    });
    volumeSeries.priceScale().applyOptions({
      scaleMargins: { top: 0.8, bottom: 0 },
    });
    volumeSeries.setData(volumeData);

    // Trade markers (v5: createSeriesMarkers, NOT series.setMarkers)
    const markersPrimitive = createSeriesMarkers(candleSeries, tradeMarkers);

    // Indicator line overlays
    indicatorLines.forEach(({ label, data }) => {
      const lineSeries = chart.addSeries(LineSeries, {
        color: '#f59e0b',  // amber-500
        lineWidth: 1,
        priceLineVisible: false,
        crosshairMarkerVisible: false,
        title: label,
      });
      lineSeries.setData(data);
    });

    // Fit content on first load
    chart.timeScale().fitContent();

    return () => {
      chart.remove();
    };
  }, [bars, trades, indicatorLines]);

  return <div ref={containerRef} style={{ height: 400 }} />;
}
```

### Pattern 2: v5 Series Creation — CRITICAL DIFFERENCE FROM v4

**What:** In v5, all series are created via `chart.addSeries(SeriesType, options)`. Individual `addCandlestickSeries()`, `addLineSeries()`, `addHistogramSeries()` methods do NOT exist in v5.

**v4 → v5 migration table:**

| v4 API | v5 API | Import needed |
|--------|--------|---------------|
| `chart.addCandlestickSeries(opts)` | `chart.addSeries(CandlestickSeries, opts)` | `import { CandlestickSeries } from 'lightweight-charts'` |
| `chart.addLineSeries(opts)` | `chart.addSeries(LineSeries, opts)` | `import { LineSeries } from 'lightweight-charts'` |
| `chart.addHistogramSeries(opts)` | `chart.addSeries(HistogramSeries, opts)` | `import { HistogramSeries } from 'lightweight-charts'` |
| `series.setMarkers([...])` | `createSeriesMarkers(series, [...])` | `import { createSeriesMarkers } from 'lightweight-charts'` |

### Pattern 3: createSeriesMarkers (v5 only)

**What:** Markers are a detached primitive in v5 — `createSeriesMarkers(series, markers)` returns a primitive instance. Updating requires calling `.setMarkers([...])` on the primitive instance (not the series).

```typescript
// Source: https://tradingview.github.io/lightweight-charts/tutorials/how_to/series-markers
import { createSeriesMarkers } from 'lightweight-charts';

// Map BacktestTrade[] to SeriesMarker[]
const markers = trades.map((t) => ({
  time: t.datetime.slice(0, 19), // ISO string truncated to seconds
  position: t.side === 'BUY' ? 'belowBar' : 'aboveBar',
  shape: t.side === 'BUY' ? 'arrowUp' : 'arrowDown',
  color: t.side === 'BUY' ? '#10b981' : '#ef4444',  // emerald-500 / red-500
  text: t.side === 'BUY' ? 'B' : 'S',
}));

const markersPrimitive = createSeriesMarkers(candleSeries, markers);
// To update later: markersPrimitive.setMarkers(newMarkers);
```

**Time format note:** lightweight-charts accepts Unix timestamps (number) or ISO date strings `"YYYY-MM-DD"` for day-based charts. For intraday 5m bars, use **Unix epoch seconds** (number). Convert `Instant` on backend to epoch seconds in the DTO, or convert ISO string on frontend: `Math.floor(new Date(isoString).getTime() / 1000)`.

### Pattern 4: Volume Histogram Overlay (single pane)

**What:** Set `priceScaleId: ''` (blank string) on histogram series to make it an overlay in the same pane. Control vertical position with `scaleMargins`.

```typescript
// Source: https://tradingview.github.io/lightweight-charts/tutorials/how_to/price-and-volume
const volSeries = chart.addSeries(HistogramSeries, {
  priceFormat: { type: 'volume' },
  priceScaleId: '',           // overlay, not left/right scale
});
volSeries.priceScale().applyOptions({
  scaleMargins: { top: 0.8, bottom: 0 }, // occupies bottom 20%
});
// Candle series pushed up to avoid overlap:
candleSeries.priceScale().applyOptions({
  scaleMargins: { top: 0.05, bottom: 0.25 },
});

// Volume data: per-bar color (green=up, red=down)
const volData = bars.map(b => ({
  time: b.time,
  value: b.volume,
  color: b.close >= b.open ? '#10b981' : '#ef4444',
}));
```

**What:** New `GET /api/v1/trading/paper/backtest/bars` on `BacktestController`. Reuses `MarketBarIntradayRepository.findBySymbolAndRange`. Returns `ApiResponse<List<CandleBarDto>>`.

```java

// New DTO in dto/CandleBarDto.java
public record CandleBarDto(
    long time,    // Unix epoch seconds (for lightweight-charts)
    double open,
    double high,
    double low,
    double close,
```

Query path: `MarketBarIntradayRepository.findBySymbolAndRange(symbol, fromInstant, toInstant)` — this method already exists and handles KST day-boundary to Instant conversion as shown in `IntradayBacktestEngine.loadBars()` (lines 167–170). The new endpoint extracts this logic into a reusable service method or calls the repository directly.

**Why BacktestController (not InternalMarketController):** `InternalMarketController` is an internal Cloud Scheduler endpoint protected by `X-Internal-Token` header. The bars endpoint must be user-facing (`/api/v1/...`) and authenticated via normal session auth from `SecurityConfig`. BacktestController at `/api/v1/trading/paper/backtest` is the natural home.

### Pattern 6: Indicator line client-side computation

**What:** Extract indicator specs from the rule definition's `entry.conditions` and `exit.conditions`. For each operand where `indicator` is `SMA` or `EMA`, compute the series using the bar close prices returned by the endpoint. This avoids a second round-trip and keeps indicator logic in the frontend where the bars are already loaded.

**Indicator extraction rule:**

```typescript
// From RuleDefinition (types/trading.ts)
function extractIndicators(def: RuleDefinition): Array<{ indicator: 'SMA' | 'EMA', period: number }> {
  const all = [
    ...(def.entry?.conditions ?? []),
    ...(def.exit?.conditions ?? []),
  ];
  const result: Array<{ indicator: 'SMA' | 'EMA', period: number }> = [];
  for (const cond of all) {
    for (const operand of [cond.left, cond.right]) {
      if ((operand.indicator === 'SMA' || operand.indicator === 'EMA') && operand.params?.period) {
        result.push({ indicator: operand.indicator, period: operand.params.period as number });
      }
    }
  }
  // deduplicate
  return [...new Map(result.map(r => [`${r.indicator}-${r.period}`, r])).values()];
}
```

Compute SMA/EMA on the close array using the same algorithm as `Indicators.java` (simple sliding window). RSI and VOLUME indicators are NOT plotted as line overlays (non-price-scale values would confuse the chart).

### Pattern 7: Row click + accordion coexistence (6.5 interaction fix)

**What:** The existing `PaperBacktestPage` handles `onClick` on `<tr>` to toggle `expandedId` (accordion). For candle chart, a separate `selectedTradeKey` state (`"${symbol}-${date}"`) is needed. Row click must BOTH set `selectedTradeKey` (to trigger chart load) AND toggle the accordion. These are independent state changes on the same event — no conflict.

**Resolution:** Single `handleRowClick(trade, index)` function that:
1. Calls `setSelectedTradeKey(...)` with the (symbol, date) of the clicked trade
2. Calls `setExpandedId(isExpanded ? null : index)` for the accordion toggle

The candle chart section is ABOVE the trade table, so the chart updates first visually before the accordion row opens below.

### Anti-Patterns to Avoid

- **Using `series.setMarkers()` directly:** This is v4 API. In v5 it does not exist on the series. Use `createSeriesMarkers(series, markers)`.
- **Using `chart.addCandlestickSeries()`:** v4 only. In v5 use `chart.addSeries(CandlestickSeries, opts)`.
- **Hardcoding hex colors:** All colors must map to tailwind.config.js tokens or known Tailwind palette values (e.g., `#10b981` = `emerald-500`, `#ef4444` = `red-500`, `#9ca3af` = `gray-400`, `#111827` = `gray-900`). Document the mapping in the component.
- **Embedding OHLCV in BacktestResult:** Locked decision says no — the endpoint is on-demand.
- **Using EmptyState/PageState with light-theme tokens (cream, charcoal) in the candle section:** Both pages use a dark theme (`bg-gray-900/50`, `border-white/10`). EmptyState component uses `bg-cream` + `text-charcoal` tokens. For the candle section, use an inline dark-themed empty state `<div>` consistent with the rest of the page rather than the generic `EmptyState` shared component (which is designed for the light-themed marketing pages).

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Candlestick rendering | Custom SVG/canvas candlestick | `chart.addSeries(CandlestickSeries)` | Handles zoom, pan, crosshair, tooltip, axis scaling automatically |
| Trade markers | Custom SVG overlay positioned manually | `createSeriesMarkers(series, markers)` | Auto-positions to bar high/low, handles zoom, scroll, scaling |
| Volume histogram layout | Custom CSS layout for subpanel | `priceScaleId: ''` + `scaleMargins` | Built-in overlay positioning with proper axis handling |
| Chart resize handling | Manual `window.resize` listener | `autoSize: true` in chart options | lightweight-charts uses ResizeObserver internally when `autoSize` is set |
| Indicator computation | Calling a separate indicator endpoint | Client-side SMA/EMA from bar close array | Bars already loaded; avoids extra round-trip; Indicators.java algorithm is trivial to replicate in TS |

**Key insight:** lightweight-charts handles all the hard parts of financial chart rendering (time scale gaps, zoom/pan state, crosshair synchronization, marker auto-positioning). Any manual override of these behaviors will break on edge cases.

## Common Pitfalls

### Pitfall 1: v4 marker API used in v5
**What goes wrong:** `series.setMarkers([...])` throws `TypeError: candleSeries.setMarkers is not a function` because the method was removed in v5.
**Why it happens:** LLMs and most online examples still reference v4 API.
**How to avoid:** Always import `createSeriesMarkers` from `'lightweight-charts'` and use `createSeriesMarkers(series, markers)`.
**Warning signs:** TypeScript will show no error (the method just doesn't exist, so the call is on `any`-typed series). Test immediately after install.

### Pitfall 2: Series creation using type-specific methods
**What goes wrong:** `chart.addCandlestickSeries()` — `TypeError: chart.addCandlestickSeries is not a function`.
**Why it happens:** v4 had individual methods; v5 removed them all in favor of `addSeries(SeriesType, opts)`.
**How to avoid:** Import the series type constant (`CandlestickSeries`, `HistogramSeries`, `LineSeries`) and use `chart.addSeries(Type, opts)`.

### Pitfall 3: Time format mismatch — ISO string vs Unix epoch
**What goes wrong:** Chart renders no data or shows wrong x-axis values. lightweight-charts treats time as either a `BusinessDay` object, a date string `"YYYY-MM-DD"`, or a Unix epoch (seconds, not milliseconds).
**Why it happens:** JavaScript `Date.getTime()` returns milliseconds; lightweight-charts expects seconds.
**How to avoid:** Backend `CandleBarDto.time` field = `ts.getEpochSecond()` (seconds). Frontend: use this directly as `time: bar.time` (number).

### Pitfall 4: KST vs UTC Instant in DB query
**What goes wrong:** The bars endpoint returns no data even though bars exist in DB, because Instant comparison uses UTC but the KST-day boundaries need explicit conversion.
**Why it happens:** `market_bars_intraday.ts` is stored as UTC Instant. A KST date "2026-01-05" starts at `2026-01-04T15:00:00Z`.
**How to avoid:** Follow the exact pattern in `IntradayBacktestEngine.loadBars()` lines 167–168:
```java
Instant fromInstant = date.atStartOfDay(ZoneId.of("Asia/Seoul")).toInstant();
Instant toInstant = date.atTime(23, 59, 59).atZone(ZoneId.of("Asia/Seoul")).toInstant();
```
Then call `findBySymbolAndRange(symbol, fromInstant, toInstant)`.

### Pitfall 5: Chart not resizing in flex/grid container
**What goes wrong:** Chart renders at 0px width inside a flex container.
**Why it happens:** `autoSize: true` uses ResizeObserver on the container div. If the container div has no explicit size (no `width` / `height` CSS), the chart observes 0.
**How to avoid:** Give the container `<div>` an explicit height via Tailwind (`h-[400px]` or inline `style={{ height: 400 }}`). Width can be `w-full`.

### Pitfall 6: Memory leak — chart not removed on unmount
**What goes wrong:** Multiple chart instances accumulate on component re-render (e.g., running multiple backtests in one session). Memory grows and canvas elements pile up in DOM.
**How to avoid:** Always return `() => { chart.remove(); }` from `useEffect`. Verify with React StrictMode which mounts/unmounts twice.

### Pitfall 7: EmptyState / PageState theming mismatch
**What goes wrong:** `EmptyState` uses `bg-cream text-charcoal` tokens (light theme for marketing pages). Inserted in the dark-themed backtest result area, it renders as an ugly light box.
**Why it happens:** The shared `EmptyState` was built for the light UI. Both PaperBacktestPage and PaperHistoryPage are fully dark-themed.
**How to avoid:** For the candle section, use an inline minimal dark empty state:
```tsx
<div className="flex h-[400px] items-center justify-center rounded-lg border border-white/10 bg-gray-900/50 text-sm text-gray-500">
  표시할 거래가 없습니다.
</div>
```
`ErrorBanner` and `SkeletonBlock` have the same issue (light-themed). Use dark-specific inline patterns or override via className.

### Pitfall 8: PaperHistoryPage `tradedAt` field vs BacktestPage `datetime`
**What goes wrong:** When extracting (symbol, date) from PaperHistoryPage trades to call the bars endpoint, using the wrong field name causes missing date derivation.
**Why it happens:** `PaperTradeHistoryItem` has `tradedAt: string` (ISO). `BacktestTrade` has `datetime: string`. The bars endpoint needs `LocalDate`, derived differently from each.
**How to avoid:** Derive date as `trade.tradedAt.slice(0, 10)` (history) vs `trade.datetime.slice(0, 10)` (backtest).

```java
) {
    public static CandleBarDto from(MarketBarIntraday bar) {

            bar.getHigh() != null ? bar.getHigh() : bar.getClose(),
            bar.getLow()  != null ? bar.getLow()  : bar.getClose(),

            bar.getVolume() != null ? bar.getVolume() : 0L
        );
    }
}
```

```typescript
function computeSMA(closes: number[], period: number): Array<{ time: number; value: number }> {
  const result = [];
  for (let i = period - 1; i < closes.length; i++) {
    const sum = closes.slice(i - period + 1, i + 1).reduce((a, b) => a + b, 0);
    result.push({ time: times[i], value: sum / period });
  }
  return result;
}
```
