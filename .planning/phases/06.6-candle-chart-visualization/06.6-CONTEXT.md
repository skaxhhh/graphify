# Phase 6.6: 5분봉 캔들 차트 시각화 - Context

**Gathered:** 2026-06-22
**Status:** Ready for planning

<domain>
## Phase Boundary

백테스트 결과 화면(PaperBacktestPage)에서 기존 수익 곡선 차트 **아래에**, lightweight-charts(Apache 2.0)로 선택한 **단일 종목·단일 거래일**의 5분봉 OHLCV를 캔들스틱으로 렌더링하고, 그 위에 진입(매수)/청산(매도) 시점을 마커로 표시한다. x축=세션 시간, y축=가격.

**범위 확장 (사용자 결정):** 로드맵은 백테스트 결과 페이지만 명시하나, 사용자가 **모의 거래 이력(PaperHistoryPage)에도 동일 캔들 차트 적용**을 명시적으로 요청. 두 화면은 이미 6.5 매매근거 아코디언·거래 테이블을 공유하므로 동일 CandleChart 컴포넌트를 양쪽 재사용한다.

**아웃-스코프:** 다종목 동시 표시(캔들은 단일 종목·세션), 여러 거래일 연속 캔들, 실투자/LIVE 화면, 새 지표/연산자 추가.

</domain>

<decisions>
## Implementation Decisions

### 1. 차트 대상 선택 방식
- **거래 행 클릭 연동**: 아래 거래 내역 테이블의 행을 클릭하면 해당 (종목·거래일)의 캔들 차트가 로드되고, 그 거래 지점으로 스크롤/강조된다. (6.5 아코디언 행과 공존하는 인터랙션 — planner가 클릭 vs 아코디언 토글 충돌 정리)
- **초기 상태 = 첫 거래 자동 로드**: 백테스트/이력 로드 직후, 첫 번째 거래의 종목·일자를 자동 선택해 차트를 바로 채운다. 이후 다른 행 클릭 시 그 종목·일자로 교체.
- **선택 가능 범위 = 거래가 발생한 (종목·일자) 조합만**. 유니버스 전체/무거래 세션은 선택지에 넣지 않는다.

### 2. 캔들 데이터 공급 (온디맨드 API)
- `BacktestResult`에 OHLCV를 임베드하지 않는다. (현재 미포함, 다종목×다일자 페이로드 과대)
- **온디맨드 GET 엔드포인트**: (종목, 날짜) 선택 시 `market_bars_intraday`에서 **해당 날짜의 전체 세션 5분봉 전부**를 조회해 반환한다. 거래 주변만 자르지 않고 그 날 세션 전체를 가져온다.
- 백테스트(휘발성 결과)·모의 이력(영속 paper_trades) 양쪽이 동일 (종목+날짜) → 동일 bars 엔드포인트를 호출한다.

### 3. 마커 표현 & 호버
- **2색 마커**: 매수 = emerald 상향 화살표(캔들 아래), 매도 = red 하향 화살표(캔들 위). lightweight-charts 기본 마커 API(`setMarkers`) 사용. 다크 테마 액센트와 일관.
- **호버/클릭 정보 = 가격·수량 + 6.5 매매근거 요약**: 마커 상호작용 시 체결가·수량·side에 더해 6.5의 충족 조건식+지표값 요약(예: `RSI(14)=28.3 < 30 ✓`)을 노출. 상세는 거래 테이블 아코디언(6.5) 유지.
- **클릭한 거래 마커 강조 + 스크롤**: 테이블에서 클릭한 거래의 마커를 다른 색/크게 강조하고 해당 지점이 보이도록 스크롤.

### 4. 지표선 오버레이
- **룰 지표선(SMA 등) 오버레이 추가**: 룰 정의에 사용된 이동평균선 등 지표를 캔들 위에 선으로 오버레이한다. (어떤 지표/기간을 그릴지는 룰 정의에서 파생 — research/plan에서 산정. 백엔드가 해당 종목·일자의 지표 라인 값을 계산해 제공)

### 5. 시간축 & 캔들 표현
- **표시 단위 = 클릭한 단일 거래일 한 세션** (09:00–12:00 백테스트 / 모의는 해당일 세션).
- **줌·패닝 허용**: lightweight-charts 기본 인터랙션 활성(캔들 디테일 탐색). 수익 곡선(recharts 정적)과 의도적으로 다름.
- **데이터 있는 봉만 연속 배치**: 점심·장외 등 빈 구간은 시간 갭 없이 봉만 이어 붙인다(lightweight 기본 동작).
- **거래량 서브패널 추가**: 캔들 아래 거래량 히스토그램 서브패널(lightweight histogram series).

### 6. 빈/로딩/에러 상태 (디자인 게이트 4상태)
- **초기**: 첫 거래 자동 로드(위 §1). 자동 로드할 거래가 0건이면 shared `EmptyState`("표시할 거래가 없습니다" 류).
- **로딩**: 온디맨드 bars fetch 중 **스피너** 표시.
- **에러/없음 구분**: 조회 **실패 = `ErrorBanner` + 재시도**, 해당 종목·일자 **데이터 없음 = `EmptyState`** 별도 문구.
- **섹션 노출**: 캔들 차트 섹션은 **항상 펼쳐서 표시**(접이식 아님). 미선택 시 영역 내 EmptyState.

### Claude's Discretion
- 차트 높이/타이포그래피/정확한 색 토큰 (tailwind.config.js 토큰만, 하드코딩 hex 금지)
- bars 엔드포인트 정확한 경로·쿼리 파라미터 형태, 응답 DTO 필드 구성
- SMA 등 지표선을 백엔드 계산 vs 프론트 계산 중 어디서 산출할지, 어떤 지표·기간을 그릴지 룰 정의에서 파생하는 규칙
- 거래 행 클릭(차트 로드)과 6.5 아코디언 토글(근거 펼침) 인터랙션 충돌 해소 방식
- 마커 호버 vs 클릭 중 정보 노출 트리거, 강조 마커 스타일
- lightweight-charts 다크 테마 옵션 매핑(배경/그리드/업·다운 캔들 색)

</decisions>

<specifics>
## Specific Ideas

- 멘탈 모델: "거래 테이블에서 한 건을 누르면, 그 거래가 어느 캔들 패턴에서 나왔는지 차트로 바로 본다"
- 마커 호버에서 6.5 근거(`RSI(14)=28.3 < 30 ✓`)를 그대로 재사용 — 차트와 테이블이 같은 근거 언어를 쓰게
- 청산 사유(익절/손절/지표)는 6.5에서 이미 구분되어 있으니 호버 요약에 반영 가능(마커 색 자체는 매수/매도 2색 유지)
- lightweight-charts의 줌·패닝을 살려 단기 모멘텀 진입/청산 캔들을 확대 관찰하는 게 핵심 사용 시나리오

</specifics>

### Reusable Assets
- `frontend/src/pages/trading/paper/PaperBacktestPage.tsx` — 폼 → 메트릭 5카드 → 수익곡선(EquityCurveChart) → 고급통계 3카드 → 거래 테이블(TradeRationaleRow) 구조. 수익곡선 섹션 아래에 캔들 섹션 삽입.
- `frontend/src/components/backtest/EquityCurveChart.tsx` — 수익곡선 차트(recharts). 캔들은 별도 `CandleChart` 컴포넌트로 신규(`components/backtest/`), recharts 아님 lightweight-charts.
- `frontend/src/components/trading/TradeRationaleRow.tsx` + `parseRationale` — 6.5 매매근거 파서/렌더. 마커 호버 근거 요약에 재사용.
- `BacktestResult.TradeDto(datetime, symbol, companyName, side, qty, price, pnl, rationaleJson)` (`backend/.../trading/dto/BacktestResult.java`) — 마커 좌표·근거의 데이터 소스. side=BUY/SELL, datetime, price 보유.
- 프론트 타입 `BacktestTrade`(`types/trading.ts`): `datetime·symbol·companyName·side·qty·price·pnl·rationaleJson` 이미 존재 → 마커 매핑에 그대로 사용.
- `backend/.../market/MarketBarIntraday.java` — `symbol·ts(Instant)·interval("5m")·open·high·low·close·volume·source`. 캔들 OHLCV + 거래량 서브패널 데이터 소스.
- `backend/.../market/MarketBarIntradayRepository.java` — 5분봉 조회 리포지토리. bars 엔드포인트가 (symbol, date, time range)로 조회.
- `backend/.../trading/backtest/IntradayBarCacheService.java` — 분봉 DB 캐시 저장/조회 서비스. bars 조회 경로 재사용 가능.
- `IntradayBacktestEngine.loadBars(symbol, date, timeFrom, timeTo)` — DB 캐시 우선, 미스 시 Yahoo fallback. bars 엔드포인트가 동일 로직 재활용 가능.
- shared 프리미티브: `EmptyState`/`ErrorBanner`/`SkeletonBlock`/`PageState` (frontend/src/components/shared/) — 4상태에 필수 사용.

### Established Patterns
- React Query: `useMutation`(runBacktest) + `useQuery`. bars 온디맨드는 `useQuery`([symbol, date]) 패턴.
- 다크 테마: gray-800/900, emerald-600 액센트, border-white/10, rounded-lg 카드. lightweight-charts 옵션을 이 토큰에 맞춰 매핑.
- API: `ApiResponse<T>` 래퍼, Spring Boot 3.4.5 / Java 21. `@/lib/ruleApi`/`apiClient` 경유.
- 차트 라이브러리 분리: recharts(수익곡선/정적) + lightweight-charts(캔들/인터랙티브) 공존. lightweight-charts는 package.json 미설치 → 신규 설치 필요.

### Integration Points
- **신규 백엔드 엔드포인트**: GET (종목, 날짜) → 해당일 전체 세션 5분봉 List<OHLCV>. `InternalMarketController` 또는 `BacktestController` 인근에 추가(planner 결정). market_bars_intraday 조회.
- **지표선 데이터**: 룰 정의에서 지표(SMA 등) 파생 → 해당 종목·일자 라인 값 계산. 엔드포인트 응답에 포함 vs 별도 (planner 결정).
- **신규 프론트 컴포넌트**: `components/backtest/CandleChart.tsx` (lightweight-charts). PaperBacktestPage + PaperHistoryPage 양쪽 마운트.
- **PaperHistoryPage 적용**: 모의 거래 이력 행 클릭 → 같은 bars 엔드포인트 호출(paper_trades의 symbol+ts에서 종목·일자 도출).
- lightweight-charts 신규 npm 설치 (package.json 미존재).
